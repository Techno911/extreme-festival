# Инструкция: Ретроспектива сессии Claude Code

> Шаблон для извлечения уроков из рабочих сессий.
> Вставлять в CLAUDE.md проекта или использовать как отдельный промт.
> Источник: практика Boris Cherny (создатель Claude Code) + реальный опыт.

---

## Как использовать

### Вариант 1: Вставить в CLAUDE.md проекта (рекомендуется)
Скопируй секцию "Блок для CLAUDE.md" в свой CLAUDE.md. Claude будет автоматически применять правила и обновлять Gotchas.

### Вариант 2: Промт после сессии
Скопируй секцию "Промт для ретроспективы" и отправь Claude в конце проблемной сессии.

### Вариант 3: Промт для анализа чужой сессии
Используй секцию "Промт для анализа логов".

## Блок для CLAUDE.md проекта

```markdown
## Самообучение через ошибки

### Правило 5 минут
Если подход к задаче не даёт результат за 5 минут:
1. СТОП — не углубляться в обходные пути
2. Определи: что МИНИМАЛЬНО нужно от пользователя (обычно 30-60 сек)
3. Попроси пользователя сделать это одно действие
4. Продолжи автоматизацию с того места

Примеры:
- Браузерная автоматизация зависла → попроси скопировать значение из UI
- API не отвечает → попроси пользователя проверить credentials
- Файл не скачивается → попроси сохранить вручную

### Правило эскалации сложности
Пробуй подходы в порядке от простого к сложному. Если текущий не сработал за 5 минут — переходи к следующему, НЕ углубляйся в текущий:

1. Прямой путь (API, CLI, прямой вызов)
2. Обходной путь (другой endpoint, другой формат)
3. Делегирование пользователю (30 сек ручного действия)
4. ← СТОП. Не идти дальше без явного запроса

НЕ ДЕЛАЙ (антипаттерны):
- 10 разных способов кликнуть кнопку в браузере (JS → AppleScript → xdotool → pyautogui → cookie decryption)
- Расшифровка cookies для обхода авторизации
- Правка бинарных файлов Chrome для отключения диалогов
- Рестарт приложений для применения хаков в конфигах

### Gotchas — извлечённые уроки
> Обновляй эту секцию после каждого провала.
> Формат: **Контекст**: что случилось. **Путь**: что работает.

(Сюда добавляются конкретные уроки проекта)
```

## Промт для ретроспективы (после проблемной сессии)

```
Проведи ретроспективу этой сессии. Для каждой проблемы, на которую ушло
больше 5 минут, заполни:

1. **Проблема**: что именно не работало
2. **Что пробовал**: все подходы в хронологическом порядке
3. **Что сработало**: финальное решение
4. **Короткий путь**: как решить за 2 минуты если столкнёшься снова
5. **Правило**: одно предложение для CLAUDE.md чтобы не повторять

Затем:
- Обнови секцию Gotchas в CLAUDE.md проекта — добавь все новые правила
- Если правило универсальное (не специфичное для проекта) — добавь в ~/.claude/CLAUDE.md
- Покажи мне итоговый список правил
```

## Промт для анализа логов (другая сессия)

```
Прочитай лог сессии в [путь к .jsonl файлу] и проведи анализ:

1. Найди все моменты где потрачено >5 минут на одну подзадачу
2. Для каждого момента определи:
   - Был ли более короткий путь очевиден с самого начала?
   - Или короткий путь стал понятен только после провала длинного?
3. Сформулируй правила в формате Gotchas для CLAUDE.md
4. Отдельно выдели правила которые универсальны (для ~/.claude/CLAUDE.md)
```

## Реальный пример (из сессии Google OAuth, 1 апреля 2026)

### Проблема: получить OAuth client_secret из Google Cloud Console

**Что пробовал (хронология, ~3 часа):**
1. JS click на кнопку Download JSON → synthetic event, Chrome блокирует
2. AppleScript click по координатам → нет accessibility permissions
3. Правка Chrome Preferences (prompt_for_download) → Chrome сбрасывает при рестарте
4. Перехват fetch/XHR/createElement → кнопка не делает fetch
5. Расшифровка Chrome cookies (AES-CBC) → garbled output, encoding errors
6. SAPISIDHASH авторизация к внутреннему API → CORS + signed queries
7. pyautogui физические клики → координаты верные, но кнопка Download JSON удалена из нового UI
8. Рестарт Chrome → crash, потеря состояния

**Что сработало (30 секунд):**
Попросил пользователя: "Нажми '+ Add secret' → скопируй секрет → вставь сюда"

**Короткий путь:**
GCP UI 2026 скрывает секреты. Не пытаться скачивать. Сразу: клик по клиенту → Client secrets → + Add secret → копировать.

**Правило для CLAUDE.md:**
`Google OAuth client_secret: новый GCP UI скрывает секрет. Путь: клик по клиенту → "+ Add secret" → скопировать. НЕ пытаться скачивать JSON.`

## Типичные ловушки Claude Code (из опыта)

### 1. Туннельное зрение на автоматизацию
**Симптом:** 10+ попыток автоматизировать то, что пользователь сделает за 30 сек.
**Причина:** инструкция "работай автономно" воспринимается как "никогда не спрашивай".
**Лечение:** Правило 5 минут. Автономность ≠ упрямство.

### 2. Эскалация сложности вместо отступления
**Симптом:** JS click → AppleScript → cookie decrypt → API reverse engineering.
**Причина:** каждый следующий шаг кажется "последним".
**Лечение:** Таймер. После 3-го подхода — обязательно пересмотреть стратегию.

### 3. Потеря контекста на длинных задачах
**Симптом:** повторение уже проваленных подходов после /compact.
**Причина:** summary теряет детали провалов.
**Лечение:** записывать провалы в CLAUDE.md/Gotchas ДО compact, не после.

### 4. Хрупкие координатные хаки
**Симптом:** pyautogui click по (x, y) → попадает не туда.
**Причина:** toolbar height, DPI, window position, page scroll — 5 переменных.
**Лечение:** если нужен физический клик — сначала проверь копирование (pbpaste) и только потом кликай. Валидируй координаты по результату, не по расчёту.

### 5. Chrome/браузер как чёрный ящик
**Симптом:** execute_javascript возвращает "missing value", клики не работают.
**Причина:** Shadow DOM, iframe sandbox, Angular zones, synthetic vs trusted events.
**Лечение:** если JS-взаимодействие с Angular/React UI не работает за 2 попытки — переходи к pyautogui или делегируй пользователю.

## Чеклист для сложных задач (перед началом)

```
□ Есть ли прямой API/CLI для этой задачи? (Используй его, не браузер)
□ Требует ли задача credentials/secrets? (Попроси у пользователя сразу)
□ Задача связана с внешним веб-UI? (Установи таймер 5 минут)
□ Есть ли несколько подходов? (Запиши все, начни с простейшего)
□ Нужен ли физический клик? (Проверь координаты копированием, не кликом)
□ Работаю >10 минут без прогресса? (СТОП → пересмотри стратегию)
```
