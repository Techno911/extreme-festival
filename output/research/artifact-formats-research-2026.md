## 1. ПИТЧ-ПИСЬМО ПАРТНЁРУ / АМБАССАДОРУ

### Проблема с .md

.md файл требует знания формата или специального приложения. Получатель (менеджер радиостанции, блогер) получит файл — увидит сырой текст с `##`, `**`, `---`.

### Рекомендация: текст в теле сообщения ТГ/email

Питч — это не документ. Это письмо. Оно живёт там, где отправляется:
- **ТГ:** текст прямо в чате. Форматирование: жирный через `*текст*`, курсив через `_текст_`
- **Email:** HTML-письмо или plain text. Google Docs позволяет скопировать отформатированный текст в Gmail без потери форматирования
- **Если нужна «визитка» к письму:** одностраничный Google Doc с шапкой фестиваля, ссылка «Только просмотр»

### Ключевые данные

| Метрика | Значение | Источник |
|---------|---------|---------|
| Сообщения в мессенджерах vs email | В 8 раз выше открываемость | workspace.ru / emailmatrix.ru, 2025 |
| Reply rate персонализированного cold email | 8–20% | Martal B2B Statistics, 2026 |
| Оптимальная длина питча | 75–150 слов | inBeat Agency, 2025 |
| Один CTA vs несколько | +371% кликов при одном CTA | Martal CTA Best Practices, 2025 |

### Вывод

Питч = текст в ТГ или в теле email. .md файл не отправляют — его используют как черновик внутри системы. Агент пишет в .md, бот доставляет в ТГ готовый текст.

## 2. МАРКЕТИНГОВЫЙ БРИФ ДЛЯ ПОДРЯДЧИКА

### Проблема с .md

Подрядчик (разработчик сайта, видеопродакшн) получает .md — не понимает форматирование, не может прокомментировать конкретный абзац, не может распечатать с нормальной разметкой.

### Рекомендация: Google Doc, ссылка «Только просмотр» / «Комментирование»

**Почему Google Doc:**
- Открывается без авторизации при настройке «Все с ссылкой — могут просматривать» (подтверждено: support.google.com/docs, 2025)
- Подрядчик может оставлять комментарии в режиме «Комментатор» — не меняя документ
- Корректно отображается на мобильном (важно для просмотра в ТГ)
- Нижняя кривая освоения: «docs.new» открывает новый документ за 2 секунды (Nuclino comparison, 2026)
- Можно скачать как PDF одним кликом для отправки по email

**Workflow:**
1. Агент генерирует бриф в .md (внутренняя версия)
2. Агент или Женя переносит в Google Doc (copy-paste, 2 минуты)
3. Женя открывает доступ «Все с ссылкой — комментатор»
4. Ссылка отправляется подрядчику

**Notion — почему нет:**
- Steep learning curve для новых пользователей (Nuclino comparison, 2026)
- Требует регистрации для комментариев
- Медленно на мобильных устройствах

| Формат | Без логина | Комментарии | Мобильный | Печать/PDF | Итог |
|--------|-----------|-------------|-----------|-----------|------|
| .md файл | да | нет | нет | нет | ✗ |
| Google Doc | да (if shared) | да | да | да | ✓ |
| Notion | нет | нет без аккаунта | медленно | да | ✗ |
| PDF | да | нет | да | да | частично |

## 3. МУДБОРД ДЛЯ ДИЗАЙНЕРА

### Проблема с .md

.md с текстом «вот референсы: [ссылка1], [ссылка2]» — это не мудборд. Дизайнеру нужно видеть картинки рядом, в одном пространстве, с возможностью сравнить.

### Рекомендация: Milanote

**Почему Milanote:**
- Read-only ссылка — **не требует регистрации для просмотра** (подтверждено на milanote.com)
- Бесплатно до 100 элементов (достаточно для мудборда из 10–20 картинок)
- Drag-and-drop, поддержка изображений, текстовых блоков, ссылок, цветовых палитр
- Экспорт в PDF высокого качества — одна кнопка
- Используется профессиональными дизайнерами (bookmarkify.io, 2026)

**Что включить в мудборд:**
- 10–20 изображений (не больше 25 — иначе «каша», NN/g 2024)
- 3–5 цветов с HEX-кодами
- 5–7 прилагательных настроения (брутальный, сырой, живой, тёмный)
- Примеры типографики
- Краткие комментарии к ключевым референсам

**Figma — почему нет:**
- Нет встроенного поиска изображений
- Нет видео
- Падает производительность при 50+ изображениях (kosmik.app, 2026)
- Не предназначен для мудбординга

**Pinterest — почему нет:**
- Хорош для сбора идей, не для финальной презентации дизайнеру
- Нет возможности добавить комментарии к референсам
- Алгоритм подмешивает нерелевантные картинки

| Формат | Без логина | Визуальность | Комментарии | PDF | Итог |
|--------|-----------|-------------|-------------|-----|------|
| .md с текстом | да | нет | нет | нет | ✗ |
| Milanote | да (read-only link) | да | да | да | ✓ |
| Figma | нет без аккаунта | да | да | да | ✗ |
| Pinterest | нет | да | нет | нет | ✗ |

## 4. КОНТЕНТ-ПЛАН

### Проблема с .md

.md таблица с 30+ строками — нечитаема на мобильном, нельзя отфильтровать по дате/платформе, нельзя быстро обновить статус.

### Рекомендация: Google Sheets

**Почему Google Sheets:**
- Открывается без логина при настройке публичного доступа
- Таблица с колонками: дата / платформа / рубрика / текст-черновик / медиа / статус / ссылка на пост
- Фильтрация: Женя видит только «Ожидает согласования» одним кликом
- Условное форматирование: красный = не готово, зелёный = опубликовано
- Работает на мобильном
- Возможен импорт данных из Google Forms (если понадобится)

**Для соло-маркетера Google Sheets > Notion/Airtable:**
- Нет learning curve
- Нет платной подписки
- Интегрируется с существующими Google-инструментами
- Airtable free tier: 1000 записей, потом $20/месяц (Airtable vs Notion, 2026)

**Структура колонок (рекомендация):**

| Дата | Платформа | Рубрика | Черновик | Медиа нужно? | Статус | Ссылка |
|------|-----------|---------|----------|-------------|--------|--------|
| 11.04 | ВК | Группа недели | [текст] | Фото | Черновик | — |

**Важно:** .md остаётся внутренним черновиком агента. В Google Sheets переносится уже согласованное или готовое к согласованию.

## 5. ТЕНДЕРНЫЙ ПАКЕТ

### Проблема с .md

Тендерный пакет состоит из нескольких разных документов: бриф (текст с логикой), скоринговая таблица (формулы), письмо подрядчику (питч). В одном .md файле — теряется структура.

### Рекомендация: Google Doc (бриф) + Google Sheets (скоринг) + текст в ТГ/email (письмо)

**Компоненты пакета:**

| Компонент | Формат | Обоснование |
|-----------|--------|-------------|
| Маркетинговый бриф (5 блоков) | Google Doc, read-only ссылка | Текст с логикой, подрядчик может комментировать |
| Скоринговая таблица | Google Sheets | Формулы автоскоринга цены, весовые коэффициенты |
| Письмо подрядчику | Текст в email / ТГ | Питч, не документ — живёт там где отправляется |
| КП от подрядчика (входящие) | Любой формат — принять | Наша инициатива не нужна |

**Почему не ZIP с PDF:**
- PDF нельзя редактировать и комментировать
- ZIP — лишний клик для открытия
- Ссылка на Google Doc открывается сразу в браузере
- PDF как финальная версия — да, но не как рабочий документ

**Стандарт HubSpot RFP (2025):** шаблоны RFP существуют в Word, Google Docs и PDF. Для активного тендера — Google Docs. PDF — только для архива финального варианта.

## 6. ИССЛЕДОВАНИЕ / АНАЛИТИКА

### Рекомендация: оставить .md + Google Sheets для данных

**Почему .md остаётся:**
- Хорошо подходит для нарратива, контекста, источников
- Версионирование через git или просто файловую систему
- Быстро читается в GitHub, VS Code, Obsidian
- Агенты работают в .md — переделывать нет смысла

**Когда нужен Google Sheets:**
- Числовые данные: цены конкурентов, охваты каналов, статистика продаж
- Сравнительные таблицы с 5+ строками
- Данные, которые будут обновляться еженедельно (мониторинг конкурентов)

**Hybrid рекомендация (Foyer/Nuclino, 2026):**
- Внутри команды / для агентов: .md
- Для Жени с числами: Google Sheets со ссылкой из .md файла
- Для подрядчиков/партнёров: Google Doc

## 7. КОНЦЕПЦИЯ ТРЕЙЛЕРА

### Проблема с .md

Концепция трейлера — это не просто текст. Это комбинация: смысловая часть (почему так, что должен чувствовать зритель) + визуальные референсы (какие сцены, какой монтаж, какой цвет).

### Рекомендация: Google Doc (концепция) + Milanote (визуальная часть)

**Google Doc — для концепции:**
- Структура: идея → нарратив → сцены → техзадание → KPI
- Видеопродакшн работает в Google Docs (Canny Creative, StudioBinder — оба рекомендуют Google Doc для video brief)
- Подрядчик может оставлять комментарии
- Экспорт в PDF для финальной версии

**Milanote — для референсов:**
- Скриншоты сцен из фильмов
- Цветокоррекция: примеры тонировки
- Настроение: 10–15 кадров
- Ссылка read-only, без логина

**Что НЕ использовать:**
- Презентация (Google Slides / PowerPoint) — создаёт иллюзию финальности, но концепция должна обсуждаться
- PDF lookbook на этом этапе — преждевременно, ограничивает правки

## 8. ЧЕКЛИСТ / ИНСТРУКЦИЯ ДЛЯ ЖЕНИ

### Проблема с .md

Женя работает в Telegram. Открыть .md файл — значит найти его в файловой системе или в боте, потом решить чем открыть. Лишние шаги = чеклист не будет использован.

### Рекомендация: Telegram-сообщение с нативным форматированием

**Формат для ТГ:**
```
Чеклист: [название задачи]

☐ Шаг 1 — конкретное действие
☐ Шаг 2 — конкретное действие
☐ Шаг 3 — конкретное действие

Дедлайн: [дата]
Файл (если нужен): [ссылка]
```

**Почему ТГ, не PDF:**
- Открываемость сообщений в мессенджерах в 8 раз выше email (emailmatrix.ru, 2025)
- Женя уже в ТГ — нулевое трение
- Можно отметить галочки в личных заметках или в боте через inline-кнопки
- PDF требует: скачать → открыть → найти чеклист — 3 лишних действия

**Когда PDF оправдан:**
- Инструкция для печати на A4 (раздать волонтёрам на фесте)
- Финальный договор с подрядчиком (архивирование)

**Данные по уведомлениям для офлайн-событий (workspace.ru, 2025):**
- Оптимальное уведомление: за 3 дня + за 1 день до дедлайна
- Telegram-сообщение с напоминанием эффективнее email-рассылки в 8 раз по открываемости

## ОБЩИЙ ПРИНЦИП: ДВУХУРОВНЕВАЯ СИСТЕМА

```
Внутри системы (агенты):       → .md файлы в output/
Доставка организатору (Женя):  → ТГ-сообщение (для действий) или ссылка Google Doc/Sheets
Доставка подрядчику:           → Google Doc (ссылка read-only/комментатор)
Доставка дизайнеру:            → Milanote (ссылка read-only)
Архив / финальные версии:      → PDF (только для договоров и финальных брифов)
```

**Правило:** .md = черновик для агентов. Никогда не отправляется напрямую организатору или подрядчику.

## РЕКОМЕНДАЦИИ ПО ПЕРЕХОДУ

### Что делать прямо сейчас (приоритет)

1. **Питч-письма** — агент генерирует текст в .md, бот выводит готовый текст в ТГ. Нулевые изменения в архитектуре.
2. **Маркетинговый бриф** — добавить шаг «создать Google Doc» после генерации .md. Женя делает copy-paste один раз.
3. **Чеклисты для Жени** — все инструкции агентов выводить через stdout в ТГ-боте с форматированием.

### Что отложить (не критично сейчас)

- Milanote: нужен только при создании мудборда для сайта/трейлера — не сегодня
- Google Sheets для контент-плана: переход когда контент-план реально заработает
- PDF: только как финальная архивная копия договора
