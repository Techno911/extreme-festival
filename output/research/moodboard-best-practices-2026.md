## 1. ТИПЫ МУДБОРДОВ

| Тип | Цель | Что включает | Когда делать |
|-----|------|--------------|--------------|
| **Visual Style** | Задать визуальный тон и эстетику | Цветовые палитры, текстуры, фотографии, типографика | До начала дизайна — первый шаг |
| **UX / Content** | Показать паттерны навигации, структуру контента | Скриншоты сайтов, wireframe-рефы, UX-паттерны | После Visual Style, перед wireframe |
| **Film / Video** | Задать тон трейлера — свет, монтаж, цветокоррекция | Кадры из фильмов/клипов, цветовые гриды, референсы операторской работы | До начала съёмок — отдельный документ от сайтового |
| **Brand** | Синхронизировать айдентику | Логотипы, шрифты, иконки, стиль иллюстраций | При работе с дизайнером айдентики |
| **Event** | Передать атмосферу мероприятия | Декор, освещение, оформление сцены, одежда гостей, фото похожих событий | При работе с декоратором или для видеосъёмки |

**Вывод для Эстрим Феста:** нужно два отдельных мудборда — (1) для сайта (Visual Style + UX), (2) для трейлера (Film). Смешивать нельзя — дизайнер сайта и режиссёр трейлера читают разные вещи.

---

## 2. ОПТИМАЛЬНОЕ КОЛИЧЕСТВО РЕФОВ

| Этап | Кол-во | Логика |
|------|--------|--------|
| Сбор (Pinterest / Behance) | 40–60 изображений | Широкая воронка, собираем всё что цепляет |
| Редактура (первый фильтр) | 20–25 изображений | Убираем дубли и не попавшие в тему |
| Финальный мудборд для подрядчика | **8–12 изображений** | Золотой стандарт: каждый реф несёт уникальную нагрузку |
| Фильм-мудборд для трейлера | **15–25 кадров** | Больше т.к. охватывает: свет, грейдинг, монтаж, движение камеры |
| «Анти-мудборд» | 5–7 изображений | Показывает что точно не нужно — снимает разночтения |

**Правило:** если два изображения говорят одно и то же — оставь одно. Каждый реф должен добавлять новую деталь, которой нет в других.

## 3. КАК АРТКОН ОБОСНОВЫВАЕТ КАЖДЫЙ РЕФ

Стандарт аннотации для каждого изображения в мудборде:

```
[Изображение / ссылка на источник]

ЧТО НРАВИТСЯ: [1–2 конкретных элемента — не "красиво", а "диагональная компоновка текста +
               монохромный фон с высоким контрастом"]
ЧТО БРАТЬ: [конкретно — "цветовую гамму", "тип шрифта", "скорость монтажа", "угол камеры"]
ЧТО НЕ БРАТЬ: [если есть элементы которые не подходят — "не брать карусель слайдов"]
ПОЧЕМУ ЭТОТ РЕФ: [связь с нашей задачей — "целевая аудитория похожая, та же эмоция входа"]
```

**Принцип «без контекста — Роршах»:** мудборд без аннотаций — тест Роршаха. Каждый видит своё. Подрядчик воспринимает один реф буквально, заказчик видел там только атмосферу. Аннотации убирают этот зазор.

**Что делать с подрядчиком:** объяснять не «скопируй это», а «меня здесь цепляет [X] — вот почему это работает для нашей аудитории». Референс = точка разговора, не техзадание.

| **Milanote** | Бесконечный холст, комментарии, ссылки, пароль-защита, нет аккаунта у зрителя | Платный для больших проектов | Финальный мудборд для подрядчика |

## 5. МУДБОРД ДЛЯ САЙТА — ЧТО ПЕРЕДАВАТЬ ПОДРЯДЧИКУ

### Структура пакета для веб-разработчика:

```
1. Visual Style Moodboard (8–12 рефов с аннотациями)
   - Общая атмосфера: 2–3 сайта с похожим вайбом
   - Типографика: 2–3 примера текстовых блоков которые нравятся
   - Цветовая палитра: HEX-коды + 2–3 скрина использования
   - Графика / иконки / иллюстрации: стиль и характер

2. UX/Content Moodboard (5–8 рефов)
   - Навигация: пример хедера/меню который нравится
   - Структура страницы: пример лайнап-блока, расписания, купи билет
   - Мобильная версия: хотя бы 1 реф адаптива

3. Анти-мудборд (5 рефов)
   - Что точно не нужно: корпоративный стиль, AI-гладкость, "вылизанность"

4. Текстовый бриф (1–2 стр.)
   - Одно предложение: "сайт должен чувствоваться как..."
   - Аудитория и что они должны почувствовать
   - 3 слова-маркера (для Эстрим Феста: жёстко / живо / без понтов)
```

### Что НЕ делать:
- Не отправлять просто папку с PNG без подписей
- Не говорить «сделай как Awwwards» — слишком абстрактно
- Не смешивать рефы сайтов разных жанров без объяснений

## 6. МУДБОРД ДЛЯ ТРЕЙЛЕРА — ОТЛИЧИЯ ОТ САЙТОВОГО

| Параметр | Сайт-мудборд | Видео-мудборд |
|----------|-------------|---------------|
| Формат рефов | Скриншоты, PNG | Видеоклипы (тайм-код!), кадры с описанием |
| Главное что передаём | Цвет, шрифт, компоновка | Ритм монтажа, движение камеры, грейдинг |
| Тайм-код | Не нужен | Обязателен: "смотри 0:32–0:47 — вот темп" |
| Цветовая палитра | HEX + примеры | LUT-референс или "грейдинг как в клипе X" |
| Звук | Не применимо | Если есть — указать BPM, жанр, настроение |
| Количество рефов | 8–12 | 15–25 кадров из разных источников |
| Анти-мудборд | Желательно | Критически важен — режиссёры любят свой стиль |

### Структура видео-мудборда для режиссёра трейлера:

```
1. Атмосфера / настроение (3–5 кадров)
   - Как ДОЛЖЕН чувствоваться зритель через 30 сек просмотра
   - Пример: кадры концертов, толпа, пот, лица в экстазе

2. Операторская работа (3–5 рефов)
   - Тип движений камеры: ханхелд? слайдер? кран?
   - Крупные планы vs общие? Угол съёмки?

3. Монтаж (2–3 рефа с тайм-кодом)
   - Темп: быстрый клиповый или дышащий?
   - Переходы: жёсткий кат или motion-blur?

4. Цвет / грейдинг (2–3 примера)
   - Тёплый / холодный / контрастный?
   - Ссылки на конкретные клипы с похожим грейдингом

5. Что НЕ делать (3–5 примеров)
   - Глянцевое промо с корпоративным звуком
   - AI-рендеры толпы
   - Таймлапсы и дроны без контекста
```

## 7. СТРУКТУРА МУДБОРДА ДЛЯ ИВЕНТ-САЙТА И ФЕСТИВАЛЯ

Конкретная структура для Эстрим Феста:

### Блок A: Атмосфера (3 рефа)
Сайты металлических фестивалей или концертных клубов с тёмной эстетикой.
Цель: показать подрядчику «вот в каком пространстве мы живём».
Источники: Awwwards → поиск «music festival», Behance → «metal band website»

### Блок B: Типографика (2 рефа)
Жирные гротески, без засечек, брутальные капсы.
Что брать: ритм текстовых блоков, иерархия H1/H2.
Что не брать: скрипты, рукописные шрифты.

### Блок C: Цвет (1–2 рефа)
Тёмный фон + 1–2 акцентных цвета (красный, жёлтый, оранжевый, белый).
Формат: конкретные HEX + скрин применения.

### Блок D: Лайнап-блок (2 рефа)
Как другие фестивали показывают список групп.
Что брать: читаемость, иерархия хедлайнер → остальные.

### Блок E: CTA «Купить билет» (1 реф)
Кнопка, её расположение, срочность.
Цель: агрессивная видимость без попсовости.

### Блок F: Анти (5 рефов)
Что точно не нужно: вылизанный корпоратив, градиенты "поп-музыка", AI-стоковые фото.

## 8. ИСТОЧНИКИ ВДОХНОВЕНИЯ: КАК ПОЛЬЗОВАТЬСЯ

| Платформа | Что искать | Как собирать | Фильтр для металла |
|-----------|-----------|--------------|---------------------|
| **Awwwards** | Awards → Music/Entertainment + Filter by Dark theme | Сохранять URL + скрин + аннотацию | Искать: «concert», «festival», «underground» |
| **Behance** | Поиск «metal band identity», «festival website» | Смотреть полные кейсы — виден процесс | Искать: «dark branding», «aggressive typography» |
| **Dribbble** | Быстрые визуальные снипеты, тренды | Скринить понравившееся, группировать по теме | Искать: «dark UI», «event landing page» |
| **Land-book** | Готовые лендинги целиком | Фильтр по индустрии / цвету | Фильтр: Dark |
| **Siteinspire** | Минимализм, типографика | Поиск по тегу | Тег: dark, music |
| **Pinterest** | Широкий сбор на ранней стадии | Отдельные доски по теме | «Metal festival poster», «dark concert website» |

### Алгоритм работы с Awwwards (рекомендован как главный источник для сайтов):

1. Зайти в Awards → Site of the Day / Site of the Month
2. Фильтр: Industry = Entertainment / Music
3. Открыть 5–7 сайтов, просмотреть 30 сек каждый
4. Скрин + URL + 1 предложение «что зацепило»
5. Перейти в Developer Tools — посмотреть шрифты
6. Сохранить в Milanote с аннотацией

## 9. РОЛЬ АРТ-ДИРЕКТОРА VS ЗАКАЗЧИКА

| Этап | Арт-директор | Заказчик |
|------|-------------|---------|
| Первичный сбор | Собирает 40–60 рефов из профессиональных источников | Присылает 3–5 примеров «мне нравится вот это» |
| Редактура | Фильтрует до 8–12, убирает дубли, добавляет аннотации | Смотрит и говорит «да / нет / почему» |
| Анти-мудборд | Предлагает примеры «что не нужно» | Подтверждает или добавляет своё «точно не так» |
| Финальная сборка | Собирает в инструменте, добавляет слова-маркеры | Утверждает (да/нет, без деталей если доверяет) |
| Передача подрядчику | Проводит онбординг: объясняет каждый реф | Не нужен на этом этапе если мудборд аннотирован |

**Ключевое правило (из практики арт-директоров):**
Мудборд — инструмент управления ожиданиями, не творческого вдохновения.
Цель: чтобы подрядчик и заказчик «видели одно и то же» до начала работы.
Без синхронизации: подрядчик сделает «как он понял», заказчик скажет «это не то».

## 10. РЕКОМЕНДАЦИИ ДЛЯ AI-АГЕНТА (конкретный алгоритм)

Когда агент-ресёрчер собирает мудборд для Эстрим Феста:

### Шаг 1. Определить задачу
- Для сайта или для трейлера? (разные форматы!)
- Какую эмоцию должен передать продукт?
- Для Эстрим Феста: «жёстко, живо, без понтов» — 3 слова-маркера

### Шаг 2. Сбор (40–60 рефов)
- Awwwards: 10 рефов (Entertainment/Music/Dark)
- Behance: 10 рефов (metal, festival, dark branding)
- Pinterest: 20 рефов (festival poster, metal concert, dark web)
- Конкуренты: 10 рефов (сайты зарубежных метал-фестов: Download, Wacken, Hellfest)

### Шаг 3. Фильтрация
- Убрать всё «попсовое» и «корпоративное»
- Убрать дубли по смыслу
- Оставить 8–12 для финального мудборда

### Шаг 4. Аннотирование (ОБЯЗАТЕЛЬНО)
Для каждого рефа:
- Источник + ссылка
- Что нравится (1–2 конкретных элемента)
- Что брать для нашего проекта
- Что НЕ брать

### Шаг 5. Сборка
- Инструмент: Milanote (приоритет) или Figma
- Добавить анти-мудборд (5 рефов «что точно не так»)
- Добавить слова-маркеры как заголовок: «ЖЁСТКО / ЖИВО / БЕЗ ПОНТОВ»

### Шаг 6. Передача подрядчику
- Ссылка Milanote (не аккаунт — просто ссылка)
- Или экспорт PDF с аннотациями
- Обязательный сопроводительный текст: «смотри особенно на X в рефе Y»
