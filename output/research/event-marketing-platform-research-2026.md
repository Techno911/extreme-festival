## БЛОК 1: Маркетинговые платформы для ивент-индустрии

### Что реально используют фестивали

Ни одна из «больших» платформ (Bizzabo, Splash, Hopin) не создана для клубного металл-феста. Они заточены под B2B-конференции и корпоративные события. Разбор по платформам:

| Платформа | Для чего | Артефакт на выходе | Цена (2025-2026) | Применимость к Эстрим Фест |
|-----------|---------|-------------------|-----------------|--------------------------|
| **Eventbrite** | Публичные тикетинговые события, discovery | Страница события, список покупателей, базовая аналитика | Бесплатно + % с продаж | Н/Д — TicketsCloud уже используется |
| **Bizzabo** | B2B-конференции enterprise | Сайт события, email-кампании, SmartBadge, спонсорский портал | $17 999/год | Избыточно и нерелевантно |
| **Splash** | Дизайн-ориентированные event-страницы | Брендированные landing pages, список гостей, отчёты | Custom pricing | Можно изучить для лендинга если Tilda не устроит |
| **Cvent** | Enterprise-мероприятия (10 000+ гостей) | Контракты с площадкой, интеграция с CRM, AI-поиск площадок | Custom enterprise | Нерелевантно |
| **Whova** | Нетворкинг на конференциях | Мобильное приложение, алгоритм мэтчинга, lead-capture | Бесплатно + платные модули | Нерелевантно |
| **Hopin** | Виртуальные и гибридные события | Виртуальная площадка, записи, данные стендов | Бесплатный тир + платные | Нерелевантно |
| **ClickUp (Festival template)** | Планирование задач и контент-план | Kanban, Gantt, календарь, статусы задач | Бесплатно / $7/мес | Применимо для внутреннего трекинга |
| **Monday.com** | Проект-менеджмент ивента | Gantt, Kanban, дашборд, автоматизация | $9-16/мес/польз. | Применимо. AI подсказывает зависимости задач |
| **Asana** | Таск-менеджмент маркетинговых кампаний | Timeline (Gantt), Board (Kanban), Calendar | Бесплатно / $10,99/мес | Применимо |
| **TicketsCloud** | Тикетинг (уже используется) | Продажи, список гостей, QR, аналитика | Комиссия с продаж | Основная платформа — продолжать |

**Вывод:** Для Эстрим Феста стек = TicketsCloud (тикетинг) + ВКонтакте + Telegram + Google Sheets (трекинг) + AI-агенты (контент). Отдельные западные платформы не нужны.

## БЛОК 2: Какой артефакт реально полезен организатору

### Что используют vs что выглядит красиво

Индустриальные исследования и практика показывают: организатор работает с конкретными артефактами, а не с «системами». Ниже — матрица форматов:

| Тип задачи | Формат «красивый» | Формат «рабочий» | Почему рабочий лучше |
|-----------|-----------------|-----------------|---------------------|
| Пост в ВК/ТГ | PDF с концепцией и описанием | Текст в .md — copy-paste сразу | Нет лишних шагов: скопировал → опубликовал |
| Письмо партнёру | Шаблон с 15 переменными | Готовое письмо + 2-3 слова для замены | Меньше думать, быстрее отправить |
| Бриф для подрядчика | Google Doc с комментариями | PDF или .md с чёткой структурой | PDF можно отправить как вложение без объяснений |
| Задачи на неделю | Kanban-доска в Notion/Asana | Список 3-5 пунктов в ТГ-сообщении | Одно сообщение утром — 30 сек на обзор |
| Аналитика продаж | Дашборд с 10 метриками | 3 числа: продано сегодня / всего / осталось до цели | Одно сообщение = понятно без кликов |
| Контент-план | Красивая таблица на месяц | Файл .md по неделям с датами и статусами | Можно дополнять, не перерисовывая |
| Исследование | Текстовый отчёт | Таблица с цифрами + выводы пулями | Цифры = проверяемый факт |

### Критерии качества артефакта (4 параметра)

**Actionability (могу ли действовать прямо сейчас?):**
- Pass: «Открой ВК → вставь текст → нажми опубликовать»
- Fail: «Этот контент создаёт вовлечённость аудитории через нарративный подход»

**Completeness (всё ли есть?):**
- Pass: письмо готово, тема указана, приветствие есть, подпись есть
- Fail: «адаптируй под контекст», «добавь детали о мероприятии»

**Format (привычный инструмент?):**
- Женя работает в Telegram и браузере. Значит: ТГ-сообщение > Google Doc > PDF > Notion
- Артефакт должен доходить туда где человек уже находится, а не требовать переключения

**Context (понятно зачем и что делать дальше?):**
- Pass: «Это письмо для Rock FM. Отправить на press@rockfm.ru. Ответ ждать 3-5 дней. Если молчат — follow-up из шаблона #2»
- Fail: «Письмо для партнёрства» (и всё)

## БЛОК 3: Критерии качества AI-артефактов

### QA-воркфлоу для AI-контента (по Search Engine Land)

Трёхфазная проверка — стандарт индустрии 2025:

**Фаза 1 — Автоматические проверки (до человека):**

| Параметр | Инструмент | Порог Pass |
|---------|-----------|------------|
| Орфография/грамматика | Grammarly / LanguageTool | 0 критических ошибок |
| Объём текста | Word count | В рамках ТЗ ±10% |
| Форматирование | Ручной просмотр | Нет лишних заголовков, нет разрыва структуры |
| Плагиат (для пресс-текстов) | Copyscape | Ниже 8% совпадений |

**Фаза 2 — Человеческая проверка (контекст):**

| Параметр | Проверяет | Вопрос |
|---------|----------|--------|
| Точность фактов | Женя или Researcher | Нет ли выдуманных цифр, дат, имён? |
| Голос бренда | Женя | Звучит ли как Эстрим Фест или как AI-пресс-релиз? |
| Тон | Женя | «Как друг в баре» или «корпоративный пресс-центр»? |
| Актуальность | Женя | Всё ли верно по состоянию на сегодня? |
| FOMO/urgency | Женя | Есть ли конкретная причина действовать сейчас? |

**Фаза 3 — Оценка:**

| Статус | Критерий | Действие |
|--------|---------|---------|
| Pass (публиковать) | Всё ОК, звучит как Женя | Публиковать |
| Revise (доработать) | Тон не тот / один факт неверный | Вернуть агенту с комментарием |
| Reject (переписать) | AI-звучание невозможно исправить | Черновик заново, другой угол |

### Definition of Done для конкретных артефактов

**Пост ВК/ТГ:**
- [ ] Объём: ВК 800-1200 зн., ТГ 200-400 зн.
- [ ] Нет слов: «уникальный», «незабываемый», «грандиозный»
- [ ] Есть конкретная дата / цифра / факт
- [ ] Есть CTA (ссылка на билеты или призыв к действию)
- [ ] Тон: разговорный, без пафоса
- [ ] Прошёл через Женю

**Письмо партнёру/блогеру:**
- [ ] Тема письма: конкретная, не «сотрудничество»
- [ ] Первый абзац: кто мы, одно предложение
- [ ] Конкретное предложение: что даём, что просим
- [ ] Контакт для ответа
- [ ] Нет вложений в первом письме (только в follow-up)
- [ ] Длина: не более 150 слов

**Исследовательский файл (researcher):**
- [ ] Итоговая таблица в начале с цифрами
- [ ] Минимум 3 источника на ключевой факт
- [ ] Все цифры в ₽ с датой актуальности
- [ ] Контакты: имя + платформа + @username или email
- [ ] Указана дата проверки

## БЛОК 4: Календарный план — мировые практики

### Структура 12-месячного плана (адаптировано)

| Горизонт | Что происходит в маркетинге | Тип активности | Формат документа |
|---------|--------------------------|---------------|-----------------|
| 12 мес. | «Save the date», анонс феста, старт продаж ранней птицы | Осведомлённость | Пост-анонс, лендинг |
| 9 мес. | Первые имена лайнапа, стройка партнёрской базы | Интерес | Посты о группах, инфо-партнёры |
| 6 мес. | Первые 50% билетов должны быть проданы к этой точке | Конверсия | Email-серия, посевы, таргет |
| 3 мес. | Финальный лайнап объявлен, старт посевов, блогеры | Ускорение | Посевы, амбассадоры, PR |
| 1 мес. | Финишный спринт, FOMO, «осталось X билетов» | Давление | Ежедневные посты, ретаргет |
| 2 недели | Behind-the-scenes, count-down, практическая инфо | Лояльность | Посты БТС, карта маршрута |
| 1 неделя | Финальная волна, розыгрыши, «завтра не будет» | FOMO-пик | Ежедневно 2-3 поста |

### Формат: Gantt vs Kanban vs Calendar

По данным Asana, Monday.com и Gupta Media:

| Формат | Для чего | Когда использовать |
|--------|---------|-------------------|
| **Gantt (Timeline)** | Видеть зависимости между задачами, критический путь | Для всей тактики а-к, тендеров, major milestones |
| **Kanban (Board)** | Статусы задач: бэклог → в работе → готово | Для еженедельных задач агентов и операционки |
| **Calendar** | Конкретные даты выхода постов и кампаний | Для контент-плана ВК/ТГ |
| **Checklist** | Пошаговые процессы без зависимостей | Для тендеров, onboarding партнёров, day-of |

**Вывод для Эстрим Феста:**
- Gantt: Google Sheets с датами по разделам а-к (уже частично есть)
- Kanban: Paperclip (уже есть)
- Calendar: Google Calendar или таблица в .md для постов
- Checklist: .md файлы по процессам (тендер, инфопартнёры)

### Что организатор видит каждое утро

Лучшая практика 2025 (по данным Ticket Fairy, Gupta Media): **утренний дайджест = одно сообщение в ТГ, 5 строк максимум:**

```
Эстрим Фест — утро [дата]
Продано: [X] билетов (всего [Y], осталось [Z] до цели 1000)
Сегодня выходит: [название поста, канал]
Задача дня: [одна конкретная задача]
Алерт: [если есть — конкурент, дедлайн, ответ партнёра]
```

## БЛОК 5: Quality Gates для AI-агентов

### Где AI работает хорошо в ивент-маркетинге (2025-2026)

| Задача | AI справляется | Человек нужен | Примечание |
|--------|--------------|--------------|------------|
| Черновик поста ВК/ТГ | Да (80% готово) | Да (финальный тон) | AI даёт структуру, Женя — голос |
| Email партнёру (первый контакт) | Да (85% готово) | Да (персонализация) | Добавить конкретику о партнёре |
| Исследование конкурентов | Да (сбор данных) | Да (выводы) | Факты собирает AI, стратегию — человек |
| Контент-план на месяц | Да (структура) | Да (темы и акценты) | AI предлагает даты и рубрики |
| FAQ для сайта | Да (90% готово) | Нет (проверить факты) | Один проход Жени на факты |
| Брифы для подрядчиков | Да (75% готово) | Да (бюджет, KPI) | Цифры всегда от Жени |
| Нативный текст от «лица группы» | Нет | Да | Голос артиста = только артист |
| Отзыв/рецензия на группу | Нет | Да | Металл-аудитория мгновенно узнаёт AI |
| Визуал, арты | Нет (запрет!) | Да (Давид, Оля) | Категорический запрет AI-визуала |
| Стратегические решения | Нет | Да | AI анализирует данные, решает Женя |

### Признаки AI-контента который провалится у металл-аудитории

По данным исследований аудитории 35-45 лет с высоким «детектором фальши»:

1. **Симметричные перечисления.** «Во-первых, во-вторых, в-третьих» — мгновенный AI-сигнал.
2. **Абстрактный пафос.** «Незабываемое музыкальное путешествие» — смерть.
3. **Отсутствие конкретики.** Нет дат, сумм, имён — значит написал не человек.
4. **Хэштеги-портянка.** #metal #festival #moscow #music — выдаёт автогенерацию.
5. **Ровный позитивный тон.** Без иронии, без «зубов» — не металл.

### Acceptance Criteria для поста (бинарная проверка)

```
Перед публикацией пост проходит следующие вопросы:
□ Если прочитать вслух — звучит как живой человек?
□ Есть хотя бы одна цифра или конкретный факт?
□ Нет ни одного из запрещённых слов (уникальный, незабываемый, грандиозный)?
□ CTA понятен — что делать прямо сейчас?
□ Женя мог бы это написать сам?

Если хоть один □ = нет → переписать
```

## БЛОК 6: Где AI-маркетинг РЕАЛЬНО работает (кейсы 2025-2026)

### Что работает на практике

**Персонализация email по сегментам:**
Фестивали, использующие AI-персонализацию писем (разный заголовок для фанатов разных жанров), показывают значительно более высокие open rate. Принцип: один и тот же анонс, но «хардкорный метал-фанат» видит текст с акцентом на лайнап, а «скейтер» — с акцентом на скейт-зону.

**Предиктивная аналитика продаж:**
Gupta Media зафиксировал U-кривую продаж для десятков фестивалей: 39% цикла = 50% билетов. AI-инструменты позволяют спрогнозировать когда будет провал и готовить mid-cycle кампанию заранее, а не реагировать постфактум.

**Автоматические chatbot-ответы на FAQ:**
Часть фестивалей использует chatbot для ответов на стандартные вопросы (как добраться, что взять, билеты). Экономия: менеджер не тратит 2-3 часа в день на одинаковые вопросы.

**Расписание сцен (AI-оптимизация):**
Крупные фестивали используют AI для минимизации конфликтов расписания (два похожих акта не должны играть одновременно). По данным MDPI (2025): планирование лайнапа сократилось с недель до дней.

**Репurposing контента:**
Интервью с артистом → AI выдаёт: 5 цитат для постов, 3 варианта email-анонса, 1 текст для пресс-релиза. Один источник → 10 артефактов.

### Что НЕ работает

| Задача | Почему AI не справляется | Что нужно делать |
|--------|--------------------------|-----------------|
| Нишевый голос | Металл-аудитория мгновенно идентифицирует AI-стиль | Только живой текст или очень плотная редактура |
| Проверка фактов о группах | AI галлюцинирует даты, состав, дискографию | Каждый факт о группах проверять вручную |
| Работа с конкретным артистом | AI не знает нюансов — кто с кем в ссоре, кто чувствителен к критике | Только Женя Косырев и живые контакты |
| Принятие решений о бюджете | AI предложит «оптимальное» без знания реальных ограничений | Решает Женя на основе AI-анализа данных |
| Органичный UGC | Нельзя попросить AI сделать «вирусный» пост из себя | Только провокация реальной реакции аудитории |
