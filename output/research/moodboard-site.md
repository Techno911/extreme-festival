# Мудборд: Сайт Эстрим Феста

> Создан: 2026-04-04
> Задача: визуальные референсы для тендера на разработку сайта
> Агент: Researcher
> Зачем: подрядчик (Tilda-студия) смотрит один раз и понимает, что делать — без звонков
> Тест готовности: дизайнер смотрит 10 минут и говорит «понял, начинаю»

### ЖЁСТКО / ЖИВО / БЕЗ ПОНТОВ

Три маркера проверки каждого решения. Если реф или элемент не проходит хотя бы один — он не подходит.

---

## ЧАСТЬ А — VISUAL STYLE (6 рефов)

Цвет, типографика, текстуры, общая атмосфера.

---

## Реф 1: Knotfest.com

**Источник:** https://www.knotfest.com
**Скриншот:** Чёрный фон, красный акцент, заголовки капсом шрифтом Bebas Neue Pro, жёсткие прямые углы везде, логотипы фестивалей как отдельные блоки.

**ЧТО НРАВИТСЯ:** Чёрно-красная палитра без компромиссов. Bebas Neue в капсе — читается как приказ, не как просьба. Border-radius: 0 — никаких скруглений, всё угловатое и жёсткое.

**ЧТО БРАТЬ:** Принцип «тёмный фон + один кровавый акцент». Bebas Neue или аналог для заголовков. Полное отсутствие скруглений в UI.

**ЧТО НЕ БРАТЬ:** Обилие рекламных блоков и премиум-подписки — у нас один продукт, не медиаплатформа.

**ПОЧЕМУ ЭТОТ РЕФ:** Аудитория Knotfest = наша аудитория. Тот же возраст, та же непереносимость корпоративного «вылизанного» стиля. Дизайн работает потому что не пытается понравиться всем.

---

## Реф 2: Wacken Open Air (wacken.com)

**Источник:** https://www.wacken.com/en/
**Скриншот:** Тёмный сайт, логотип черепа, карточки артистов с единообразными рамками-фреймами, обратный отсчёт с датой, «Sold Out» как статусный символ.

**ЧТО НРАВИТСЯ:** Карточки артистов в одинаковых рамках — создают ритм без хаоса. Обратный отсчёт в hero-секции — визуальный FOMO без слов. Логотип черепа = мгновенная идентификация жанра.

**ЧТО БРАТЬ:** Модульный grid для лайнапа — одинаковые карточки с иерархией по размеру (хедлайнер крупнее). Счётчик оставшихся мест в hero. Единый визуальный «фрейм» для всех артистов.

**ЧТО НЕ БРАТЬ:** Wacken — бренд с 30-летней историей, поэтому там можно быть более академичным. Нам нужнее энергия и драйв, меньше «музей».

**ПОЧЕМУ ЭТОТ РЕФ:** Эталон жанра. Если новый металлист ищет «как должен выглядеть сайт металл-феста» — он видит Wacken. Нам надо взять структурные решения, добавить своего характера.

---

## Реф 3: Hellfest (hellfest.fr)

**Источник:** https://www.hellfest.fr/en/
**Скриншот:** Чёрный фон, белый текст, минималистичная типографика без засечек. Лайнап: крупная карточка с фото хедлайнера + текстовый список остальных. Навигация по дням и сценам.

**ЧТО НРАВИТСЯ:** Лайнап-блок: хедлайнер с фотографией, остальные — текстом. Иерархия без пошлятины (не «ВЫ ЖДАЛИ — ОН ЗДЕСЬ!!!»). Просто: крупнее = главнее.

**ЧТО БРАТЬ:** Систему подачи лайнапа — один большой блок хедлайнера + ровный список остальных с кликабельными именами. Минималистичную навигацию без перегруза.

**ЧТО НЕ БРАТЬ:** Hellfest — 6 сцен, 4 дня. У нас — 2 сцены, 1 день. Их структура фильтров избыточна для нашего масштаба.

**ПОЧЕМУ ЭТОТ РЕФ:** Один из лучших по балансу «информативность vs атмосфера». Контент не тонет в украшениях, но и пустым не выглядит.

---

## Реф 4: Mastodon (mastodonrocks.com)

**Источник:** https://www.mastodonrocks.com
**Скриншот:** Чёрно-белый контраст, логотип Leviathan в белом на тёмном. Grid-лейаут, модульная сетка. Никаких градиентов, никаких декоративных элементов — только типографика и контраст.

**ЧТО НРАВИТСЯ:** Монохром как художественное решение, а не экономия. Логотип повторяется в хедере и футере — создаёт «рамку» страницы. Grid без украшений — контент сам по себе достаточен.

**ЧТО БРАТЬ:** Принцип «типографика + контраст = всё что нужно». Монохромный фундамент + 1 акцентный цвет поверх него.

**ЧТО НЕ БРАТЬ:** Полный монохром без акцентного цвета — у нас нет 20-летней истории группы, которая говорит сама за себя. Нам нужен один яркий акцент для CTA.

**ПОЧЕМУ ЭТОТ РЕФ:** Доказывает что тёмный минимализм работает в металл-контексте. Убирает страх «а вдруг будет пустовато?» — нет, будет брутально.

---

## Реф 5: Download Festival — лайнап-постер (downloadfestival.co.uk)

**Источник:** https://www.downloadfestival.co.uk/lineup/
**Скриншот:** Лайнап в формате постера — логотипы групп разного размера, иерархия по размеру (хедлайнеры большие, поддержка мелкая), разделение по дням и сценам.

**ЧТО НРАВИТСЯ:** Постер как самостоятельный артефакт — его можно распечатать, поставить обои, отправить другу. Иерархия понятна без объяснений: Guns N' Roses в 3 раза крупнее остальных.

**ЧТО БРАТЬ:** Идею «лайнап-постер» как отдельный блок на сайте. Иерархию через размер типографики/логотипов.

**ЧТО НЕ БРАТЬ:** Отсутствие интерактивности — только статичная картинка. Нам нужны кликабельные ссылки с инфой о группе.

**ПОЧЕМУ ЭТОТ РЕФ:** Постер — это язык металл-культуры. Металлисты выросли на афишах. Этот формат вызывает доверие.

---

## Реф 6: Фестивальный dark UI — цветовые решения

**Источник:** Паттерн из анализа Wacken + Hellfest + Knotfest
**Скриншот:** Собирательный образ: чёрный/очень тёмно-серый фон, один яркий акцент (красный #C43127 у Knotfest, оранжево-жёлтый у Wacken), белый текст, жирная гротескная типографика.

**ЧТО НРАВИТСЯ:** Тёмный фон = атмосфера концертного зала. Один акцент = фокус внимания. Белый текст на тёмном = читаемость без усилий.

**ЧТО БРАТЬ:** Принцип «1 фон + 1 акцент + белый текст». Не больше двух рабочих цветов.

**ЧТО НЕ БРАТЬ:** Красный акцент Knotfest — у нас нет ассоциации со Slipknot. Нам нужен свой цвет.

**ПОЧЕМУ ЭТОТ РЕФ:** Консенсус индустрии. Все топовые металл-фесты пришли к этой формуле не случайно.

## ЧАСТЬ Б — CONTENT / UX (4 рефа)

Навигация, структура блоков, подача контента, мобильная версия.

---

## Реф 7: Wacken — hero-секция с urgency

**Источник:** https://www.wacken.com/en/ (hero)
**Скриншот:** Главный экран: название феста, дата, «Final Ticket Tier», обратный отсчёт до события, CTA «Get Your Tickets Now!». Всё на первом экране без скролла.

**ЧТО НРАВИТСЯ:** Первый экран отвечает на 3 вопроса сразу: что? (фест), когда? (дата), что делать? (купить). Обратный отсчёт — визуальный FOMO без копирайта «успей купить».

**ЧТО БРАТЬ:** Структуру hero: название + дата + место + обратный отсчёт + одна кнопка «Купить билет». Всё на первом экране, ничего лишнего.

**ЧТО НЕ БРАТЬ:** Sold Out статус — нам рано (и надеюсь не понадобится). Обилие партнёрских логотипов в hero — у нас пока мало партнёров.

**ПОЧЕМУ ЭТОТ РЕФ:** UX-паттерн из лучших. Первый экран = конверсионная машина, не обложка журнала.

---

## Реф 8: Hellfest — лайнап с фильтрами

**Источник:** https://www.hellfest.fr/en/lineup/
**Скриншот:** Крупный блок хедлайнера с фото + имя + «Stage 1». Ниже: текстовый список остальных по алфавиту, все кликабельны. Фильтр по дням сверху (кнопки-таблетки).

**ЧТО НРАВИТСЯ:** Два уровня детализации — быстрый обзор (кто главный) + полный список (все имена). Фильтры не перегружают, но дают контроль.

**ЧТО БРАТЬ:** Двухуровневую структуру лайнапа. Фильтр по сценам (у нас 2: основная + летняя) как переключатель сверху.

**ЧТО НЕ БРАТЬ:** Четырёхдневную навигацию — у нас один день.

**ПОЧЕМУ ЭТОТ РЕФ:** Лайнап — главная страница сайта для нашей аудитории. Они приходят смотреть «кто играет», а не читать «о фестивале». UX должен это учитывать.

---

## Реф 9: Download Festival — «Artist A-Z» страница

**Источник:** https://www.downloadfestival.co.uk (структура навигации)
**Скриншот:** Отдельная страница с полным алфавитным списком всех артистов. Каждая ссылка ведёт на карточку группы: фото, биография 3-5 строк, ссылки на соцсети/стриминги.

**ЧТО НРАВИТСЯ:** Каждая группа — отдельная страница. Это SEO (Google находит запрос «[группа] Москва 2026») + контент для аудитории, которая хочет узнать незнакомое имя.

**ЧТО БРАТЬ:** Страницу каждой группы: фото + 3-5 строк текста + ссылки. Не «карусель на главной», а отдельные URL.

**ЧТО НЕ БРАТЬ:** Алфавитный список на 200 групп — у нас 13. Достаточно grid 4×3.

**ПОЧЕМУ ЭТОТ РЕФ:** «Искатель нового» (AJTBD сегмент 5) — незнакомые группы. Карточка группы = его точка входа. Без неё 30% аудитории уйдёт.

---

## Реф 10: Knotfest — мобильная версия

**Источник:** https://www.knotfest.com (mobile)
**Скриншот:** Стiky-хедер с логотипом и гамбургером. Hero — полноэкранное изображение + крупный заголовок + кнопка. Лайнап — карточки в одну колонку. CTA везде видны.

**ЧТО НРАВИТСЯ:** Тёмный стиль не теряется на мобиле. Sticky CTA «Купить» — всегда доступна при скролле. Изображения оптимизированы под вертикальный экран.

**ЧТО БРАТЬ:** Sticky CTA-кнопку «Купить билет» на мобиле. Полноэкранный hero на телефоне (атмосфера не теряется). Одна колонка для лайнапа на мобиле.

**ЧТО НЕ БРАТЬ:** Тяжёлые JS-анимации — на слабых телефонах не работают.

**ПОЧЕМУ ЭТОТ РЕФ:** 60-70% нашей аудитории придёт с телефона. Если мобильная версия не работает — половина конверсии потеряна.

## ЧАСТЬ В — АНТИ-МУДБОРД (7 рефов)

Что точно НЕ делать. Примеры из реальных сайтов.

---

## Анти-реф 1: PARK LIVE (parklivefest.ru)

**Источник:** http://parklivefest.ru/
**Почему нет:** Тёмно-синий (#0d6db4) акцент — это корпоративная синева, не металл. Белый фон с синими кнопками = банк, не фестиваль. Аудитория 35-45 лет мгновенно скажет «это не наше».

**Ключевая ошибка:** Цвет выбран нейтральный, «чтобы всем понравилось». Нашей аудитории нравится именно неоднозначное.

---

## Анти-реф 2: AI-сгенерированные металл-постеры (Midjourney-стиль)

**Источник:** Любой пост с промптом «metal festival poster AI»
**Почему нет:** Аудитория видит AI-арт мгновенно. Слишком идеальные блики, слишком симметричные черепа, слишком «правильная» текстура металла. Дизайнер Давид запрещает категорически — и он прав.

**Ключевая ошибка:** Визуальная фальшь = недоверие. Металлисты 35-45 лет, которые ходят на концерты 20 лет, чувствуют это подкожно.

---

## Анти-реф 3: Корпоративный «ивент» лейаут

**Источник:** Стандартные шаблоны Tilda для корпоративных мероприятий
**Почему нет:** Светлый фон, пастельные акценты, округлённые углы, иконки из Flaticon, раздел «Наши спикеры» с круглыми аватарами. Это конференция по продажам, не металл-фест.

**Ключевая ошибка:** Neutral + safe = невидимый. Для нашей аудитории «безопасный» дизайн = сигнал что организаторы не понимают культуру.

---

## Анти-реф 4: Поп-фестивальный стиль (неоновые градиенты)

**Источник:** Шаблоны типа «Электронный фестиваль» с неоновыми фиолетово-розовыми градиентами
**Почему нет:** Фиолетовый (#7C3AED), розовый, cyan — цвета EDM и поп. Металлисты ассоциируют это с «попсой» и уходят.

**Ключевая ошибка:** Цвет = сигнал жанра ещё до прочтения текста. Неоновый градиент говорит «здесь играет не ваша музыка».

---

## Анти-реф 5: Автогенерированный «официальный» стиль

**Источник:** Сайты концертных агентств с шаблонной вёрсткой
**Почему нет:** Раздел «О нас», «Наши ценности», «Команда», «FAQ» — бюрократическая структура для тех кто боится. Металлисты хотят: кто играет, когда, сколько стоит, как добраться. Всё.

**Ключевая ошибка:** Структура сайта говорит о приоритетах организатора, а не о потребностях покупателя.

---

## Анти-реф 6: Перегруженный декором «metal aesthetic»

**Источник:** Любительские сайты с черепами, цепями, кожаными текстурами поверх фото
**Почему нет:** Декоративные черепа, готические шрифты (Olde English), текстуры ржавчины и кожи — это китч. Аудитория 35+ видит это как «школьник обклеил скетчбук».

**Ключевая ошибка:** Металл-эстетику создаёт качество исполнения, а не количество черепов.

---

## Анти-реф 7: Тяжёлые анимации без цели

**Источник:** «Award-winning» промо-сайты с parallax на каждом блоке
**Почему нет:** Parallax на телефоне = тормоза. Анимация ради анимации = время до первого клика растёт. Наш пользователь пришёл купить билет, а не смотреть шоу на сайте.

**Ключевая ошибка:** Анимация = инструмент акцента, не цель. Один entrance-эффект на hero — да. Весь сайт в motion — нет.

## ТИПОГРАФИКА

**Заголовки (H1, H2):** Bebas Neue / Anton / Impact-класс — капс, жирный, без засечек. Читается как крик, не просьба.

**Тело текста:** Inter / Roboto / IBM Plex Sans — нейтральный гротеск. Читаемый, не мешает.

**Принцип контраста:** заголовки ОГРОМНЫЕ (80-120px на десктопе), текст нормальный (16-18px). Нет «средних» размеров — только контраст.

**Никаких:** скрипт-шрифтов, готики, декоративных начертаний.

## СТРУКТУРА СТРАНИЦ

### Главная (одностраничник):
1. **Hero:** Название феста + дата (11 июля, Москва) + место + счётчик дней + кнопка «Купить»
2. **Лайнап:** Хедлайнер (большой блок) + все 12 групп (grid 4×3)
3. **О фестивале:** 3-4 строки текста + 2-3 факта цифрами (13 групп, 2 сцены, 1400 мест)
4. **Программа:** Расписание по сценам (таймлайн)
5. **Скейт-зона:** Отдельный блок — мост для неметалл-аудитории
6. **Билеты:** Цены + TicketsCloud виджет + FAQ
7. **Партнёры:** Логотипы (Ernie Ball + будущие)
8. **Контакты:** Один email + соцсети

### Карточка группы (отдельная страница):
- Фото/арт группы
- Название + жанр + год основания
- 3-5 строк текста без пафоса
- Ссылки: ВКонтакте, Spotify/YouTube

## ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

- **Платформа:** Tilda (приоритет) или кастом на React/Vue
- **Адаптив:** Mobile-first, 320px → 1440px
- **Шрифты:** Google Fonts или локальные (не Adobe Fonts — лицензия)
- **Анимации:** Только entrance-эффекты на hero и лайнап-блок. Всё остальное — без motion.
- **Время загрузки:** < 3 сек на мобиле с 4G
- **SEO:** title + description + OG-теги для каждой страницы группы
- **Аналитика:** Яндекс.Метрика + пиксель ВКонтакте
