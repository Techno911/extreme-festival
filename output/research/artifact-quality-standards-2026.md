## 1. МАРКЕТИНГОВЫЙ БРИФ: стандарты качества

### Что такое хороший бриф

Бриф — это "единственный источник правды для всех, кто работает над проектом" (HubSpot, 2025). Он заменяет первый созвон с подрядчиком. Если подрядчик после прочтения брифа звонит с базовыми вопросами — бриф плохой.

**Ключевой тест:** дай бриф незнакомому исполнителю. Если через 30 минут он не задал ни одного вопроса по существу задачи — бриф хороший.

### Плохой бриф vs хороший бриф

| Критерий | Плохой бриф | Хороший бриф |
|----------|-------------|--------------|
| Задача | «Сделай сайт для фестиваля» | «Лендинг на Tilda, 5 секций, билеты через TicketsCloud, мобильная версия, сдать к 15.05» |
| ЦА | «Металлисты, 18–45 лет» | «Треш-металлисты 35–45, ненавидят AI-контент, платят 3500₽ за концерт, обитают в ВК-группах…» |
| Бюджет | «Бюджет есть» | «Бюджет на разработку: 80–120 тыс. ₽. Предоплата 30%» |
| KPI | «Хотим продать билеты» | «1000 продаж к 1.07, конверсия лендинга ≥3%, CPL ≤350₽» |
| Тон | «Современный дизайн» | «Тёмный металл-стиль, без пастельных цветов, без AI-арта» |

Источник (TrinityP3 podcast, 2024): "Many marketing teams expect the agency to write their own brief or just fill out a form. A quick phone call is not a brief."

### Обязательные 5 элементов

1. **Контекст** — 3–5 предложений о проекте. Что это, почему сейчас, что уже сделано
2. **Задача** — конкретный результат: файл/страница/кампания + дата сдачи
3. **Целевая аудитория** — 2–4 сегмента с демографией, болями и местами обитания
4. **KPI** — цифры: охват, конверсия, CPL, количество. Без «стать лучше»
5. **Ограничения** — бюджет в ₽, запрещённые форматы, обязательный стиль, красные линии

### Объём и структура

- Бриф для подрядчика (сайт, видео): **5–15 страниц**
- Бриф для блогера/амбассадора: **1–2 страницы**
- Питч-письмо с кратким брифом: **1 страница A4 максимум**

Источник: HubSpot Creative Brief Guide (2025), Funnel.io "Brief for Marketing Agencies" (2024)

### Definition of Done — маркетинговый бриф

- [ ] Есть конкретный файл/ссылка на результат (не «контент», а «лендинг tilda.cc/extremefest»)
- [ ] Написана ЦА с болями, не просто демография
- [ ] Указан бюджет в рублях с диапазоном
- [ ] KPI — конкретные цифры, не «увеличить охват»
- [ ] Указаны красные линии и запреты (например: без AI-арта, без Roboto)
- [ ] Указан дедлайн сдачи
- [ ] Текст проверен: нет слов «уникальный», «профессиональный», «качественный»
- [ ] Объём: 5–15 страниц для подрядчика, 1–2 для блогера

## 2. ПИТЧ-ПИСЬМО: стандарты качества

### Ключевые данные по результативности (2025)

| Метрика | Средний показатель | Хороший | Отличный |
|---------|-------------------|---------|---------|
| Open rate холодного письма | 27.7% | 35%+ | 45%+ |
| Reply rate холодного письма | 3.43% (платформы) / 5.1% (реальный) | 10–15% | 15%+ |
| Response rate персонализированного | 8–20% (база) | 30%+ при точной персонализации | — |

Источник: Martal B2B Cold Email Statistics 2026, Instantly.ai Benchmarks, SalesCaptain 2025

### Оптимальная длина

- **Холодное письмо инфлюенсеру/медиапартнёру:** 75–100 слов (inBeat Agency, 2025)
- **Питч партнёрства/инфопартнёрства:** 100–150 слов максимум
- **Тема письма:** 4–7 слов, без кликбейта

Источник: Cleverly.co Cold Email Best Practices 2025–26

### Что работает vs что не работает

| Работает | Не работает |
|---------|------------|
| Упоминание конкретного поста/контента получателя | «Ваша аудитория нам очень подходит» |
| Первое предложение = факт о них, не о нас | «Мы крутой фестиваль треш-металла» |
| Один CTA (конкретный вопрос или предложение) | Несколько вариантов «позвоните или ответьте или посетите сайт» |
| Тон: «написано за 5 минут от руки» | Официальный корпоративный тон |
| Персонализация: имя + ссылка на конкретный контент | Шаблонное «уважаемый коллега» |

**Данные A/B тестов:** письма с одним CTA получают на 371% больше кликов чем с несколькими CTA. (Martal CTA Best Practices, 2025)

### Структура питч-письма (5 элементов)

```
1. Зацепка (1 предложение) — что-то конкретное об их контенте
2. Кто мы (1–2 предложения) — без пафоса, с цифрой
3. Что предлагаем (1–2 предложения) — конкретно, с выгодой для них
4. Один CTA — простой следующий шаг
5. Подпись — имя + контакт
```

### Стратегия follow-up

- 58% ответов — на первое письмо
- 2–3 follow-up увеличивают суммарные ответы на ~50%
- Интервал: 3–7 дней между касаниями
- Максимум: 3 касания, потом стоп

Источник: SalesCaptain Cold Email Statistics 2025

### Definition of Done — питч-письмо

- [ ] Тема письма: 4–7 слов, без «сотрудничество», «предложение», «уникальная возможность»
- [ ] Первое предложение — о получателе, не о нас
- [ ] Длина: 75–150 слов
- [ ] Один CTA (конкретный вопрос, не «свяжитесь с нами»)
- [ ] Персонализировано: имя + конкретная деталь об их работе
- [ ] Нет корпоративных слов: «взаимовыгодное сотрудничество», «уникальное предложение»
- [ ] Есть имя и контакт отправителя
- [ ] Прочитано вслух — звучит как человек, не как PR-релиз

## 3. КАЛЕНДАРНЫЙ ПЛАН ДЛЯ ФЕСТИВАЛЯ: стандарты качества

### Фазы маркетинга по временным промежуткам

Данные по музыкальным фестивалям (TicketFairy 2024–25, Guidebook 2024):

| Фаза | Срок до события | Ключевые действия | Артефакты |
|------|----------------|-------------------|-----------|
| Стратегическая | 6+ месяцев | Сайт, ранняя продажа, save the date | Лендинг, контакты СМИ |
| Анонс лайнапа | 3–4 месяца | Reveal групп, питч блогерам | Пресс-кит, питч-письма |
| Активная раскрутка | 2–3 месяца | Посевы, таргет, посты группы | Контент-план, рекламные креативы |
| Нагнетание | 4–6 недель | FOMO, счётчик, акции | Тексты FOMO, UTM-кампании |
| Финальный спурт | 2 недели | High-frequency retargeting, логистика | Напоминальные письма, FAQ |
| День события | 11.07 | Фото, контент в реальном времени | — |
| После | +1–2 дня | Спасибо, опрос, итоги | Письмо, highlights |

### Данные о продажах билетов

Из исследования по фестивалям:
- **15% билетов** продаётся в первую неделю после открытия продаж
- **24% билетов** продаётся в последние 7 дней перед событием
- **61% билетов** продаётся в промежутке — через тиражируемые анонсы и рекламу

**Вывод:** нельзя «включить рекламу за 2 недели». Равномерная работа весь период критична.

Источник: TSE Entertainment Festival Timeline Guide, Weeztix Ticket Sales Strategy

### Уровни детализации

**Milestone-план (стратегический):** 1 строка = 1 дата. Для Жени на холодильник.
```
01.05 — открыть продажи 2 волны (3500₽)
15.05 — анонс 3 групп лайнапа
01.06 — запустить таргет ВК
15.06 — анонс иностранного хедлайнера
28.06 — финальный спурт, FOMO
11.07 — фест
```

**Недельный план (операционный):** что делать на этой неделе. Таблица: задача + дедлайн + ответственный.

**Дневной план (контент-план):** дата + платформа + рубрика + текст + статус.

### Definition of Done — календарный план

- [ ] Все даты до 11.07 внесены
- [ ] Каждый milestone привязан к ответственному (даже если это один Женя)
- [ ] Продажи билетов явно видны как отдельный трек (не смешаны с контентом)
- [ ] Указан бюджет по фазам (хотя бы диапазон)
- [ ] Есть пометки зависимостей (например: «таргет запускается только после готовности лендинга»)
- [ ] Мобильно читаемо: одна таблица, не 10 листов

## 4. KANBAN ДЛЯ МАРКЕТИНГА: лучшие практики

### Колонки доски

Канонический вариант (Atlassian, 2025):
```
Бэклог → В работе → На проверке → Не принято → Готово
```

Для команды из 1 человека (Женя) допустимо упрощение:
```
Бэклог → В работе → На проверке → Готово
```

**Что в каждой колонке:**

| Колонка | Что здесь | Критерий перехода |
|---------|----------|------------------|
| Бэклог | Сформулированные задачи без исполнителя | SMART написан |
| В работе | Задача взята в работу | Исполнитель выбран, предыдущая закрыта |
| На проверке | Результат готов, ждёт утверждения | Файл создан и доступен |
| Не принято | Возвращено с комментарием | Написано что именно исправить |
| Готово | Утверждено. Закрыто | Женя сказал «ок» или Critic дал OK |

### WIP-лимиты

- Данные Kanban University (2024): команды с WIP-лимитами улучшают delivery time на **37%**
- Правило: WIP ≤ количество человек + 1
- Для маркетинга из 1 человека: **1 задача в работе максимум**
- Для Review колонки: **2–3 задачи** (можно несколько ждут одновременно)

Источник: Atlassian WIP Limits Guide, AgileSherPas WIP Limits for Agile Marketing

### SMART-формулировка задачи

**Чеклист SMART:**
- Specific: чёткий результат (файл, страница, список, черновик)
- Measurable: можно проверить без звонка («черновик ВК-поста 800–1000 зн.»)
- Achievable: один человек за 1–3 дня
- Relevant: двигает к продажам билетов или тактике
- Time-bound: конкретная дата

**Антипаттерны формулировок:**

| Плохо | Хорошо |
|-------|--------|
| «Написать пост про группу» | «Черновик поста ВК (800–1000 зн.) про Master. Файл: output/drafts/2026-05-05-master.md. До 05.05 пт 17:00» |
| «Исследовать конкурентов» | «Таблица: Скрежет vs Эстрим Фест — цены, лайнап, подписчики. Файл: output/research/competitor-2026-05-05.md. До 05.05» |
| «Сделать бриф» | «Маркетинговый бриф для тендера на сайт (5 блоков, 5–15 стр.). Файл: output/tactic/б-сайт-бриф.md. До 10.05» |

### Definition of Done — Kanban-задача

- [ ] Название: глагол + конкретный результат
- [ ] Файл на выходе указан (output/папка/имя-файла.md)
- [ ] Дедлайн: конкретная дата
- [ ] Ответственный: агент или «Женя»
- [ ] Если 3+ шагов — есть чек-лист пунктов
- [ ] WIP-лимит соблюдён: предыдущая задача закрыта перед взятием новой

## 5. DEFINITION OF DONE ПО ТИПАМ АРТЕФАКТОВ

### 5.1 Маркетинговый бриф — когда "готов"

**Тест:** отдай бриф незнакомому подрядчику. Если через 30 минут нет базовых вопросов — готов.

Чеклист DoD:
- [ ] Контекст: 3–5 предложений о проекте (что это, зачем, что уже сделано)
- [ ] Конкретная задача: что именно нужно создать + формат
- [ ] ЦА: 2–4 сегмента с болями и местами обитания
- [ ] KPI: цифры (не «улучшить», а «1000 продаж», «CPL ≤350₽»)
- [ ] Бюджет в рублях с диапазоном
- [ ] Дедлайн сдачи
- [ ] Красные линии (что нельзя)
- [ ] Объём: 5–15 страниц
- [ ] Нет слов «уникальный», «качественный», «индивидуальный»

---

### 5.2 Питч-письмо — когда "готово"

**Тест:** прочитай вслух. Если не стыдно послать другу — отправляй.

Чеклист DoD:
- [ ] Длина: 75–150 слов
- [ ] Тема: 4–7 слов, персонализирована
- [ ] Первое предложение: о получателе, не о нас
- [ ] Один конкретный CTA (не «свяжитесь с нами когда удобно»)
- [ ] Персонализация: имя + одна конкретная деталь об их работе
- [ ] Нет «взаимовыгодного сотрудничества» и «уникального предложения»
- [ ] Есть подпись: имя + контакт
- [ ] Есть план follow-up: когда и что написать если нет ответа

---

### 5.3 Мудборд — когда "готов"

**Тест:** покажи Давиду (дизайнеру). Если он говорит «понял настроение» — готов.

Данные: мудборды прямо влияют на доверие заказчика. «Когда стейкхолдеры видят качественный мудборд, они ожидают такого же уровня в финальном дизайне» (NN/G, 2024).

Чеклист DoD:
- [ ] 10–20 визуальных референсов (не меньше 10, не больше 25 — иначе каша)
- [ ] Все изображения высокого разрешения (не пикселированные скриншоты)
- [ ] Выделена цветовая палитра (3–5 цветов с кодами)
- [ ] Добавлены слова-прилагательные: «брутальный», «живой», «тёмный», «сырой»
- [ ] Есть примеры типографики (если нужна)
- [ ] Нет противоречащих друг другу референсов (если пастель рядом с метал-тематикой — убрать)
- [ ] Комментарий: почему этот референс, что именно нравится

---

### 5.4 Контент-план — когда "готов"

**Тест:** content-ops может начать писать посты не задавая вопросов — план готов.

Из Sprout Social (2025) и Hootsuite (2025): контент-план должен содержать дату, платформу, рубрику, текст/описание, медиа и статус.

Чеклист DoD:
- [ ] Покрывает минимум 4 недели вперёд
- [ ] Для каждого поста: дата + платформа + рубрика
- [ ] Для каждого поста: текст или описание контента (не просто «пост про группу»)
- [ ] Указано нужное медиа (фото/видео/нет)
- [ ] Статус каждого поста: черновик / на согласовании / согласован / опубликован
- [ ] Баланс рубрик: не более 40% одной рубрики
- [ ] UTM-метки для постов с продажей билетов
- [ ] Каждый пост, идущий в публикацию — прошёл через Женю

---

### 5.5 Тендерный пакет — когда "готов"

**Тест:** незнакомый подрядчик может прислать полноценное КП без звонка — пакет готов.

Данные: «более ⅓ брендов получают неполные или неточные ответы на тендеры» (Superside, 2024). Причина — плохой тендерный пакет.

Чеклист DoD:
- [ ] Описание компании/проекта: кто, что, когда, масштаб
- [ ] Scope of Work (SOW): что именно нужно создать, в каком формате, сколько итераций
- [ ] Описание ЦА: кто конечный потребитель
- [ ] KPI: как будет оцениваться результат
- [ ] Бюджет: диапазон или максимум в рублях (без бюджета — неполный тендер)
- [ ] Сроки: дата запуска, промежуточные сдачи, финальный дедлайн
- [ ] Критерии оценки: как будете выбирать (по чему, с какими весами)
- [ ] Контакт для вопросов
- [ ] Формат ответа: что должен прислать подрядчик в КП
- [ ] Объём: 10–30 страниц (или структурированный Google Doc)

## 6. ПРИМЕНЕНИЕ К ЭСТРИМ ФЕСТУ

### Почему организатор не понимает пользы артефактов

Три главные причины (из анализа источников и контекста проекта):

1. **Артефакт не самодостаточен.** Бриф без KPI — не бриф, питч без персонализации — спам. Если отдать подрядчику и он не может начать работу — артефакт не готов.

2. **Нет Definition of Done.** Когда нет чёткого чеклиста «готовности», артефакт генерируется как «текст на страницах», а не как «инструмент работы».

3. **Артефакт не привязан к действию.** Бриф должен идти подрядчику сегодня. Питч — отправляться сегодня. Мудборд — показываться Давиду сегодня. Если нет немедленного следующего шага — артефакт воспринимается как «ещё один файл».

### Рекомендуемый стандарт: каждый артефакт = карточка Kanban

```
Артефакт: Питч-письмо для Rock FM
Статус: На проверке
Файл: output/research/питч-блогеры.md
Следующий шаг: Женя проверяет → отправляет на radio@rockfm.ru до 10.04
DoD пройдён: [x] 150 слов [x] персонализировано [x] один CTA [x] подпись
```
