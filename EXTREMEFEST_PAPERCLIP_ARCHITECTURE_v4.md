## 4. ГОЛОСОВОЙ ПАЙПЛАЙН (Voice → Validate → Recommend)

### Как работает:

```
Артём наговаривает голосовое в Telegram
         │
         ▼
Telegram Bot (Notifier) → Whisper → текст
         │
         ▼
Создаётся Issue в Paperclip: "Идея от Артёма: [текст]"
         │
         ▼
CEO распределяет → Critic берёт задачу
         │
         ▼
Critic валидирует в 3 ролях:
  🎸 Метал-скуф 40 лет: «Это зайдёт тусовке?»
  📊 Маркетолог: «Это повлияет на билеты?»
  🎪 Организатор фестивалей: «Это реализуемо одним человеком?»
         │
         ▼
3 раунда критики → финальный вердикт:
  ✅ «Нормально — делаем» + конкретные шаги
  ⚠️ «Есть нюансы» + что доработать
  ❌ «Не стоит» + почему + альтернатива
         │
         ▼
Результат → Telegram Артёму
```

### Промт для Critic:

```markdown
Ты получил идею от организатора фестиваля. Оцени её в 3 ролях:

РОЛЬ 1 — Металлист 40 лет, ходит на концерты 20 лет:
- Это зайдёт тусовке? Не фальшь ли это?
- Не будет ли это «вылизанным» контентом, который мы ненавидим?

РОЛЬ 2 — Маркетолог с опытом event-индустрии:
- Это повлияет на продажу билетов? Прямо или косвенно?
- ROI: сколько стоит → сколько принесёт (примерно)
- Есть ли риски (бюджет, сроки, репутация)?

РОЛЬ 3 — Организатор фестивалей (опыт 5+ мероприятий):
- Реализуемо ли это одним человеком (Женя)?
- Успеваем ли до 11 июля?
- Что может пойти не так?

После 3 ролей: дай ФИНАЛЬНЫЙ ВЕРДИКТ (✅/⚠️/❌) и конкретные шаги.
```

### ★ DESK AJTBD: ИССЛЕДОВАНИЕ АУДИТОРИИ ПО ЗАМЕСИНУ (без интервью)

**Зачем:** Без понимания «работ» аудитории вся тактика — стрельба вслепую. AJTBD даёт причинно-следственную связь: почему человек купит билет, какие барьеры мешают, какие силы толкают, какое решение он «нанимает» сейчас вместо нашего фестиваля.

**Ограничение:** Классический AJTBD = 10-15 глубинных интервью по 60 мин. У нас нет ни времени, ни бюджета на это.

**Решение: Desk AJTBD — четырёхэтапный пайплайн:**

```
ЭТАП 1: Сбор сырых данных (Researcher)
         │
         ▼
ЭТАП 2: AI-симуляция JTBD-интервью (Researcher + Critic)
         │
         ▼
ЭТАП 3: Синтез — сегменты, граф работ, job stories (Writer)
         │
         ▼
ЭТАП 4: Валидация организаторами (Женя + Женя Косырев → правки)
```

|---|---|---|

| **YouTube-комменты** под клипами групп из лайнапа | Что пишет аудитория: какие эмоции, какие ожидания от живых выступлений | Web search |

#### ЭТАП 2: AI-СИМУЛЯЦИЯ JTBD-ИНТЕРВЬЮ (Researcher + Critic)

На основе собранных данных Researcher создаёт **5 виртуальных респондентов** (AI-персоны), каждый представляет гипотетический сегмент:

| Персона | Описание | Гипотеза Big Job |
|---|---|---|
| **Серёга, 42** | Метал-скуф, ходит на концерты 20 лет, знает все составы, работает инженером. Москвич | «Почувствовать себя частью тусовки, которая понимает меня» |
| **Даша, 24** | Знает Rammstein и Metallica, татуировки, скейтборд, ищет новый опыт. Москва | «Найти свою тусовку и попробовать что-то жёсткое/новое» |
| **Антон, 38** | Из Рязани, фанат Master и Арии, ездит на концерты в Москву 2-3 раза в год | «Вырваться из рутины и получить заряд на месяцы» |
| **Макс, 30** | Музыкант, играет в метал-группе из среднего эшелона, друзья участвуют в фесте | «Поддержать своих + посмотреть как делают большие ивенты» |
| **Олег, 50** | OG-металлист, помнит «Металлоплаку» 90-х, ходит всё реже. Скептик | «Убедиться, что метал ещё жив и стоит моего времени» |

**Процесс:** Researcher проводит **симулированное JTBD-интервью** с каждой персоной, используя гайд Замесина:
1. Расскажи, когда ты в последний раз ходил на метал-ивент? (контекст + триггер)
2. Что ты ожидал получить? (работа / желаемый результат)
3. Что ты чувствовал до/во время/после? (эмоциональная работа)
4. Какие альтернативы ты рассматривал? (конкурирующие решения)
5. Что могло бы тебя остановить? (барьеры / силы против)
6. Что бы тебя точно убедило пойти? (силы за / триггеры покупки)

**Critic** проверяет каждое симулированное интервью: «Серёга 42 года реально бы так ответил? Или это AI-фантазия?»

**Результат Этапа 2:** 5 заполненных JTBD-канвасов (по одному на персону).

#### ЭТАП 3: СИНТЕЗ (Writer)

Writer собирает из 5 канвасов:

**1. JTBD-сегменты (не демографические, а по работам):**

| Сегмент | Big Job | Core Jobs | Текущее решение |
|---|---|---|---|
| «Свой в тусовке» | Почувствовать принадлежность к метал-комьюнити | Увидеть знакомые лица, обсудить группы, выпить с друзьями | Клубные концерты 50-150 чел. |
| «Искатель нового» | Найти свою субкультуру / получить новый опыт | Послушать что-то жёсткое вживую, увидеть скейтеров, купить мерч | YouTube + Spotify + фестивали других жанров |
| «Паломник из региона» | Вырваться из рутины, получить заряд | Доехать, отрываться весь день, уехать вечером с историями | Редкие поездки в МСК на концерты |
| «Музыкант-инсайдер» | Поддержать тусовку + посмотреть организацию | Увидеть коллег на сцене, потусить бэкстейдж, обменяться контактами | Собственные концерты + чужие выступления |
| «Скептик-ветеран» | Убедиться, что метал ещё стоит внимания | Увидеть легенд (Master), оценить звук/организацию, сравнить с 90-ми | Ничего — сидит дома, «всё уже не то» |

**2. Граф работ фестиваля (иерархия):**

```
Big Job: «Получить уникальный опыт метал-фестиваля»
│
├── Core Job: «Послушать живой мощный звук»
│   ├── Small Job: «Увидеть любимую группу»
│   ├── Small Job: «Открыть новую группу»
│   └── Small Job: «Почувствовать физику звука (бас, громкость)»
│
├── Core Job: «Быть частью тусовки»
│   ├── Small Job: «Встретить знакомых»
│   ├── Small Job: «Познакомиться с новыми людьми»
│   └── Small Job: «Чувствовать что я среди своих»
│
├── Core Job: «Получить впечатления за рамками музыки»
│   ├── Small Job: «Посмотреть скейтеров»
│   ├── Small Job: «Купить мерч (идентичность)»
│   ├── Small Job: «Поесть/попить на месте»
│   └── Small Job: «Сделать фото/видео для соцсетей»
│
└── Core Job: «Решить логистику»
    ├── Small Job: «Купить билет (не переплатить)»
    ├── Small Job: «Добраться (регионы — электричка)»
    └── Small Job: «Успеть уехать домой (до 22:00)»
```

**3. Силы прогресса (Forces of Progress) для каждого сегмента:**

| Сила | Пример для «скептика-ветерана» |
|---|---|
| **Push (толкает от текущей ситуации)** | «Давно никуда не ходил, скучно, друзья зовут» |
| **Pull (тянет к фестивалю)** | «Master играют! Иностранный хедлайнер! Такого давно не было» |
| **Привычка (держит в текущей ситуации)** | «Да ну, опять будет как обычно: плохой звук, толпа, пиво по 500₽» |
| **Тревога (мешает сделать шаг)** | «А вдруг организация говно, как всегда? Деньги на ветер» |

**4. Ключевые сообщения по сегментам (для контента, сайта, постов):**

| Сегмент | Ключевое сообщение | Какой барьер снимает |
|---|---|---|
| «Свой в тусовке» | «12 групп, 2 сцены, все свои» | Привычка: «и так хожу на концерты» → но здесь 12 за день! |
| «Искатель нового» | «Метал + скейт + мерч + еда. Один день — целый мир» | Тревога: «я не знаю эти группы» → не только про группы |
| «Паломник из региона» | «3 часа на электричке, завершение в 22:00, успеешь домой» | Логистика: главный барьер для регионов |
| «Скептик-ветеран» | «Master + иностранный хедлайнер. Звук от [Макс]. Не как обычно» | Тревога: «организация будет говно» → нет, тут профи |

#### КАК AJTBD ПРОНИЗЫВАЕТ ВСЮ ТАКТИКУ

| Раздел тактики | Как использует AJTBD |
|---|---|
| **(а) Стратегия** | Сегменты = основа позиционирования. Не «метал-фестиваль для всех», а 5 сегментов с разными работами |
| **(б) Сайт** | Структура лендинга строится по графу работ: каждый блок закрывает Core Job. Сообщения — по Forces of Progress |
| **(в) Трейлер** | Сценарий показывает выполнение работ всех сегментов: тусовка, скейт, мерч, звук, атмосфера |
| **(г) Амбассадоры** | Амбассадор обращается к своему сегменту: скейтер → «искатели нового», метал-звезда → «скептики-ветераны» |
| **(д) Инфопартнёры** | Рок-радио → «свои в тусовке», городские паблики → «искатели нового», региональные → «паломники» |
| **(е) Блогеры** | Леос (500к) → «музыканты-инсайдеры» + «свои в тусовке». Скейт-блогеры → «искатели» |
| **(ж) Контент** | Каждый пост адресован конкретному сегменту с конкретным сообщением. Не «мы крутые», а «твоя работа будет выполнена» |
| **(к) Календарный план** | Волны маркетинга привязаны к Forces: сначала Pull (лайнап, хедлайнеры), потом снятие Anxiety (организация, звук, логистика) |

**⚠️ Опасение:** Desk AJTBD без реальных интервью — это гипотеза, не факт. Качество зависит от: (1) насколько хорошие сырые данные соберёт Researcher, (2) насколько честно Critic проверит AI-симуляции, (3) насколько глубоко организаторы дадут обратную связь. Это не замена полноценного исследования, но для первого фестиваля с бюджетом 0₽ на ресёрч — лучшее из возможного.

#### В.3 ICE-ПРИОРИТИЗАЦИЯ (CEO)

CEO ранжирует каждого по ICE:

| Критерий | Что оценивается |
|---|---|
| **Impact** | Сколько людей из нашей ЦА увидят рекомендацию |
| **Confidence** | Шанс на согласие: есть ли связь через Женю Косырева? Знакомый? Холодный контакт? |
| **Ease** | Сложность: «Написать в DM» vs «Найти PR-агентство → письмо → месяц ожидания» |

Результат: ранжированный список. Первым пишем тем, у кого ICE максимален.

**Результат:** Ранжированный список с ICE-баллами. Работаем сверху вниз.

**UTM-метки:** Writer готовит инструкцию: каждому блогеру/каналу — уникальная ссылка `extremefest.org?utm_source=[канал]&utm_medium=[формат]`. Без UTM — нельзя понять, кто реально привёл билеты.

Notifier → Telegram Жене: «На этой неделе опубликовано X из Y запланированных посевов. Охват: Z».

**Critic** валидирует каждый нативный текст перед отправкой блогеру/каналу: «Скуф 40 лет увидит это в ТГ-канале — фальшь или живой текст?»

### ФАЗА С: ДЕПЛОЙ НА СЕРВЕР (когда сервер выбран)

> **Требования к серверу:** европейский VPS (Hetzner / DigitalOcean / Contabo), 4 GB RAM, 30+ GB диск. Российский IP несовместим с Anthropic API.

#### Шаг 0С: Создать сервер (5 мин)
```bash
# Выбрать провайдера (Hetzner рекомендуется: €5-7/мес, Финляндия/Германия)
# Добавить SSH-ключ при создании
# Зафиксировать IP → прописать в DNS: ef.chirkov.info и ef-bot.chirkov.info → A-запись
```

#### Шаг 1С: Подготовка сервера (30 мин)
```bash
ssh root@<EU_SERVER_IP>
# Swap 2GB (обязательно — нет swap = OOM при нагрузке агентов)
fallocate -l 2G /swapfile && chmod 600 /swapfile
mkswap /swapfile && swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
# Node.js + pnpm + claude-code
apt update && apt upgrade -y
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
npm install -g pnpm@9 @anthropic-ai/claude-code
```

#### Шаг 2С: Claude авторизация на сервере (10 мин)
```bash
claude login   # → URL в терминале → открыть на локальном Mac → авторизоваться
claude -P      # persistent mode
```

#### Шаг 3С: HTTPS для webhook + защита UI (20 мин)
```bash
# Получить SSL сертификаты (certbot на хосте или acme.sh)
certbot certonly --standalone -d ef-bot.chirkov.info -d ef.chirkov.info
# Настроить nginx reverse proxy:
#   ef.chirkov.info → localhost:3100 (+ basic auth htpasswd)
#   ef-bot.chirkov.info/telegram/webhook → localhost:3001
```

#### Шаг 4С: Скопировать проект с Mac на сервер (5 мин)
```bash
rsync -av ~/extremefest/ root@<EU_SERVER_IP>:/opt/extremefest/
```

#### Шаг 5С: Telegram webhook вместо polling (5 мин)
```bash
# В .env: USE_POLLING=false, WEBHOOK_URL=https://ef-bot.chirkov.info/telegram/webhook
# Зарегистрировать webhook в Telegram:
curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://ef-bot.chirkov.info/telegram/webhook"
```

#### Шаг 6С: Systemd сервисы (автозапуск при reboot)
```bash
# /etc/systemd/system/extremefest-paperclip.service
# /etc/systemd/system/extremefest-notifier.service
systemctl enable extremefest-paperclip extremefest-notifier
systemctl start extremefest-paperclip extremefest-notifier
```

#### Шаг 7С: Backup в crontab
```bash
crontab -e
# Добавить:
0 3 * * * tar -czf /opt/backups/extremefest_$(date +\%Y\%m\%d).tar.gz /opt/extremefest/output/ 2>/dev/null
```

#### Шаг 8С: Финальный smoke test на сервере
```bash
curl https://ef.chirkov.info          # → Paperclip UI (запрос пароля)
curl https://ef-bot.chirkov.info      # → 200 OK
# В UI: /check-status → дайджест прогресса (перенесённые Issues видны)
```
