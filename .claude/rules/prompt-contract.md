# Prompt Contract — обязательный стандарт для КАЖДОЙ задачи агенту

## Принцип
Каждая задача, отправляемая агенту (через ceo-create-subtask.js или напрямую), ОБЯЗАНА содержать Prompt Contract. Без него задача НЕ берётся в работу.

## Формат Prompt Contract

```
ЗАДАЧА: [глагол + конкретный результат]

ЗАЧЕМ: [1 предложение — как это помогает продать 1000 билетов к 11 июля]

КРИТЕРИИ ПРИЁМКИ:
- [ ] Формат: [md / Google Sheets / текст для ТГ]
- [ ] Объём: [конкретные цифры: 800-1200 знаков / 5-15 страниц / таблица 10+ строк]
- [ ] Содержание: [что ОБЯЗАТЕЛЬНО включить]
- [ ] Ограничения: [что запрещено]
- [ ] Файл: [точный путь output/outreach/ambassadors/имя.md]

ТЕСТ ГОТОВНОСТИ: [одно предложение — как проверить что артефакт готов]
Пример: «Женя может скопировать текст и отправить в ТГ без правок»

КОНТЕКСТ: [ссылки на зависимости — другие файлы, предыдущие задачи]
```

## Тест готовности по типам артефактов

| Артефакт | Тест готовности |
|----------|----------------|
| Питч-письмо | Женя копирует → вставляет в ТГ → отправляет. Без правок. |
| Маркетинговый бриф | Подрядчик читает → начинает работу → НЕ звонит с базовыми вопросами |
| Черновик поста ВК | Женя копирует → вставляет в ВК → публикует. Хештеги на месте. |
| Мудборд | Дизайнер смотрит → говорит «понял настроение» → начинает работу |
| Тендерный пакет | Подрядчик получает → присылает КП → без дополнительных вопросов |
| Исследование | Читатель за 30 сек понимает: что нашли, что это значит, что делать |

## Traceability — каждый артефакт содержит шапку

```markdown
> Создан: [дата]
> Задача: [CMP-XX — название]
> Агент: [кто создал]
> Зачем: [1 строка]
> Тест готовности: [пройден / не пройден]
> Следующий шаг: [что делать с этим артефактом]
```

## Information Degradation — правило передачи

Когда Head-агент создаёт subtask для Specialist:
1. Оригинальный запрос пользователя включается ДОСЛОВНО (в кавычках)
2. Head добавляет контекст, но НЕ перефразирует запрос
3. Acceptance criteria пишутся Head-агентом, а не копируются из запроса
