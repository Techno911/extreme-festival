# Мудборд-коллекция по стандарту арт-директора

## Когда использовать
- Подготовка к тендеру на сайт (раздел б)
- Подготовка к тендеру на трейлер (раздел в)
- Любая задача, где нужно передать подрядчику визуальное видение

## Два отдельных мудборда (НИКОГДА не смешивать)

### Мудборд 1: Сайт (Visual Style + Content/UX)
**Для кого:** веб-дизайнер/студия на Tilda
**Структура:**

**Часть А — Visual Style (5-6 рефов)**
Что задаёт: цвет, типографика, текстуры, атмосфера, эмоция

**Часть Б — Content/UX (3-4 рефа)**
Что задаёт: навигация, структура блоков, как подаётся контент, анимации

**Часть В — Анти-мудборд (5-7 рефов)**
Что задаёт: что точно НЕ делать. Снимает 80% разночтений.

**Итого: 13-17 рефов** (8-10 основных + 5-7 анти)

### Мудборд 2: Трейлер (Film)
**Для кого:** режиссёр/видеограф
**Структура:**

**Часть А — Визуальный тон (5-7 кадров)**
Что задаёт: цветокоррекция, свет, контраст, грейдинг

**Часть Б — Монтаж и движение (5-7 кадров с таймкодами)**
Что задаёт: темп монтажа, движение камеры, переходы

**Часть В — Атмосфера/звук (3-5 кадров)**
Что задаёт: настроение, энергия, ритм

**Часть Г — Анти-мудборд (5-7 кадров)**
Что задаёт: что точно НЕ снимать

**Итого: 18-26 кадров** (13-19 основных + 5-7 анти)

## Формат аннотации каждого рефа (ОБЯЗАТЕЛЬНО)

```
## Реф [N]: [Название/Источник]

**Источник:** [URL или "Behance / Awwwards / фестиваль X"]
**Скриншот:** [описание что на картинке]

**ЧТО НРАВИТСЯ:** [1-2 конкретных элемента — НЕ "красиво", а "диагональная
                  компоновка текста + монохромный фон с высоким контрастом"]
**ЧТО БРАТЬ:** [конкретно — "цветовую гамму", "тип шрифта", "скорость монтажа"]
**ЧТО НЕ БРАТЬ:** [элементы из этого рефа, которые не подходят]
**ПОЧЕМУ ЭТОТ РЕФ:** [связь с нашей задачей и аудиторией — 2-3 строки]
```

**Принцип: без аннотаций — тест Роршаха.** Подрядчик видит одно, заказчик имел в виду другое. Аннотации убирают зазор.

## Три слова-маркера для Эстрим Феста

Ставятся заголовком каждого мудборда:
### ЖЁСТКО / ЖИВО / БЕЗ ПОНТОВ

Каждый реф проверяется: «Это жёстко? Это живое? Это без понтов?» — если нет, реф не подходит.

## Где искать рефы

### Для сайта:
- **Awwwards** → фильтр Entertainment + Dark theme + Scroll animation
- **Behance** → «metal band identity», «festival website», «dark web design»
- **Land-book.com** → Events/Entertainment
- **Конкуренты:** Download Festival, Wacken, Hellfest — их сайты
- **Российские:** Скрежет металла, PARK LIVE, Roof Fest — что работает, что нет

### Для трейлера:
- **Vimeo** → «music festival promo», «metal concert cinematic»
- **YouTube** → трейлеры Download, Wacken, Bloodstock
- **No Film School** → статьи о фестивальном видео
- **StudioBinder** → гайды по мудбордам для видео

## Алгоритм работы агента

1. **Собрать 30-50 рефов** (широкая воронка) из источников выше
2. **Отфильтровать** до 8-12 по уникальности (каждый реф = новая деталь)
3. **Аннотировать** каждый по формату выше
4. **Собрать анти-мудборд** — 5-7 примеров «что точно не так»
5. **Указать цветовую палитру** — 5-8 HEX-оттенков
6. **Сохранить** в output/research/moodboard-site.md или moodboard-trailer.md
7. **Передать Critic** на проверку: «это для металлистов 35-45 лет? Это жёстко/живо/без понтов?»

## Что передаётся подрядчику вместе с брифом

1. Маркетинговый бриф (стандарт ЧиП 2.5)
2. **Мудборд Visual Style** (5-6 рефов с аннотациями)
3. **Мудборд Content/UX** (3-4 рефа с аннотациями)
4. **Анти-мудборд** (5-7 рефов)
5. Цветовая палитра (5-8 HEX)
6. Три маркера: ЖЁСТКО / ЖИВО / БЕЗ ПОНТОВ

## Красные линии (для Эстрим Феста)
- НЕ предлагать AI-генерированные изображения как рефы
- НЕ использовать «вылизанный» корпоративный дизайн
- НЕ рекомендовать пастельные/мягкие палитры
- Аудитория: металлисты 35-45 лет — ненавидят фальшь
- Давид (дизайнер) категорически против AI — рефы должны быть от реальных дизайнеров/сайтов
