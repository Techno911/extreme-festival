# Алгоритм выбора рубрик — 5 шагов

> Применять при запуске нового контент-плана или пересмотре рубрик.
> Источник: ЧиП SMM-рубрикатор 3.1.

## Шаг 2: Отобрать топ-кандидатов по каждому этапу

Из каждого этапа воронки берём **1–2 сценария с наибольшим потенциалом** при **минимальном ресурсе**.

Приоритет: Потенциал ÷ Ресурс. Высокий потенциал + низкий ресурс = первым в список.

**Результат для Эстрим Феста:**

| Этап | Выбрано | S-ID | Обоснование |
|------|---------|------|-------------|
| Привлечение | Группа недели | S-01 | Потенциал 5, ресурс низкий. Готовые фото у групп |
| Привлечение | Скейт | S-16 | Отдельная ЦА, расширяет охват |
| Прогрев | Extreme Opros | S-10 | Уникальный формат, высокий ER |
| Прогрев | За кулисами | S-02 | Живой контент, не требует продакшна |
| Продажа | FOMO / Билеты | S-12 | Прямая конверсия |
| Лояльность | Скетч от Оли | S-04 | Авторский, человеческий, отличает от конкурентов |
| Гибкий | Инфоповод | — | Всегда найдётся повод |

**В резерв** (не в основной план, но готовы к ротации):
- S-09 Атмосфера, S-11 Комикс, S-14 Партнёры, S-17 Fast Food, S-21 Мерч, S-23 Таймер

## Шаг 3: Назначить дни недели

Правило: **привязка к дню НАВСЕГДА.** Аудитория привыкает к ритму. Алгоритмы ВК/ТГ тоже.

Логика распределения:
- Понедельник — старт недели, тяжёлый контент (новая группа) работает лучше
- Пятница — продажа, аудитория активнее покупает перед выходными
- Суббота — лёгкое, развлекательное (скетч, комикс)
- Воскресенье — гибкий слот, не привязывать жёстко

**Результат для Эстрим Феста:**

| День | Рубрика | S-ID | Почему этот день |
|------|---------|------|-----------------|
| Пн | Группа недели | S-01 | Старт недели = максимальное внимание |
| Вт | Extreme Opros / За кулисами | S-10 / S-02 | Прогрев в середине недели |
| Ср | Атмосфера / Партнёры | S-09 / S-14 | Поддержание интереса |
| Чт | Скейт / Fast Food / Мерч | S-16 / S-17 / S-21 | Привлечение новой ЦА (скейтеры) |
| Пт | FOMO / Билеты | S-12 / S-23 | Продажа = пятница, перед выходными |
| Сб | Скетч Оли / Комикс | S-04 / S-11 | Развлечение = выходные |
| Вс | Гибкий слот | — | Инфоповод или резервная рубрика |

## Шаг 4: Проверить баланс воронки

Считаем: сколько постов в неделю приходится на каждый этап.

```
Итого 7 постов/неделю:
  Привлечение: Пн (S-01) + Чт (S-16/S-17/S-21) = 2–3 поста = 29–43% ✅ (план 40%)
  Прогрев:     Вт (S-10/S-02) + Ср (S-09/S-14)  = 2 поста    = 29%   ✅ (план 25%)
  Продажа:     Пт (S-12/S-23)                    = 1–2 поста  = 14–29% ✅ (план 20%)
  Лояльность:  Сб (S-04/S-11)                    = 1 пост     = 14%   ✅ (план 15%)
```

Если баланс не сходится — заменить рубрику из резерва на нужный этап.

**Красный флаг:** больше 3 продажных постов в неделю = аудитория устаёт и отписывается.

## Шаг 5: Валидация через Critic и финальное утверждение

Перед запуском контент-плана с новыми рубриками:

1. **Critic проверяет** по чеклисту:
   - [ ] Каждая рубрика имеет S-ID
   - [ ] Баланс воронки 40/25/20/15 соблюдён (±5%)
   - [ ] Нет двух продажных рубрик в соседние дни
   - [ ] Есть 1 гибкий слот
   - [ ] У каждой рубрики определён маяк (лучший пост конкурента или свой)
   - [ ] Грейд проверки прописан для каждой рубрики

2. **Женя утверждает** финальную таблицу рубрик

3. **Занести в справочник** Google Sheets (колонки AB–AF листа «Контентная»)

4. **Запомнить дату старта** — именно от неё считается «минимум 4 недели до замены рубрики»

## Когда пересматривать рубрики

| Ситуация | Действие |
|---------|---------|
| Прошло 2 недели, есть данные | Двухнедельная проверка (шаг 3 SKILL.md) |
| Рубрика ранг 6–7 на протяжении 4+ недель | Переработка или замена из резерва |
| Появился новый инфоповод (хедлайнер, партнёр) | Гибкий слот или временно заменить слабую рубрику |
| Меньше 3 недель до феста | Переходить на ежедневный финальный отсчёт (S-12) |
| Рубрика требует ресурс которого нет (Оля занята) | Временно заменить на низкоресурсную из резерва |
