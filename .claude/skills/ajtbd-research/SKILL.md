# Desk AJTBD Research (по методологии Замесина)

## Цель
Построить AJTBD-профиль аудитории фестиваля без живых интервью.

## Алгоритм
1. Собрать сырые данные: ВК-комменты, форумы, YouTube, ТГ-чаты, отзывы конкурентов
2. Создать 5 AI-персон (см. @references/personas.md)
3. Провести симулированные интервью по гайду (см. @references/interview-guide.md)
4. Синтезировать: сегменты + граф работ + Forces + ключевые сообщения
5. Отправить на валидацию critic'у

## Ключевые концепции AJTBD
См. @references/zamecin-framework.md

## Gotchas
- Desk AJTBD — гипотеза, не факт. Всегда помечай как «требует валидации организаторами»
- Не выдумывай сегменты — опирайся на реальные цитаты из ВК/форумов
- Граф работ: Big Job → Core Jobs → Small Jobs (не путать уровни)

## Формат Блока 4 для маркетингового брифа (стандарт ЧиП 2.5)

При запросе на формирование Блока 4 брифа — используй один из двух форматов:

### Вариант А: Если проводилось AJTBD-исследование

Для каждого сегмента (2-4 штуки):
```
Сегмент: [Название]
- Job (работа): какую задачу клиент «нанимает» продукт решить
- Trigger: что запускает решение купить СЕЙЧАС
- Context: где, когда, при каких обстоятельствах
- Pain: что мешает купить или использовать
- Where: конкретные каналы, группы, площадки
```

### Вариант Б: Если AJTBD не проводилось

Для каждого сегмента (2-4 штуки):
```
Сегмент: [Название]
- Демография: возраст, пол, география, доход
- Боль: зачем им продукт? Какую проблему решает?
- Ситуация: когда приходят? Что произошло?
- Где обитают: конкретные каналы (ВК-группы, ТГ-каналы, YouTube)
```

### Привязка к S-ID

Каждый сегмент связывается с S-ID из `context/smm-karta-kommunikatsiy.md`:
```
Сегмент → S-ID mapping:
- «Свои в тусовке» → S-01 (Группа недели), S-04 (Бэкстейдж)
- «Скептики» → S-07 (Билеты/FOMO), S-05 (Площадка)
- «Искатели нового» → S-03 (Скейт/экстрим), S-06 (Еда/атмосфера)
```

Это позволяет каждому посту точно попадать в нужный сегмент аудитории.
