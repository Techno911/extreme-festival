# Definition of Done — полный чеклист переходов

> Стандарт ЧиП Kanban 2.4. Переход между колонками разрешён ТОЛЬКО при выполнении всех пунктов.

---

## Бэклог → В работе

Проверяет: **исполнитель** перед тем как взять задачу.

- [ ] Название задачи содержит глагол + конкретный результат
- [ ] Задача отвечает SMART: S — конкретная, M — измеримая, A — за неделю одним человеком, R — связана с целями феста, T — дедлайн указан явно
- [ ] Дедлайн: конкретная дата + время (по умолчанию пятница 17:00)
- [ ] Ответственный назначен (агент или Женя Аржевский)
- [ ] Известен артефакт на выходе (файл, таблица, текст, решение)
- [ ] Если в задаче 3+ действий — чек-лист написан
- [ ] Если задача > 2 дней — разбита на подзадачи, каждая со своим SMART
- [ ] WIP-лимит соблюдён: у исполнителя нет другой задачи «В работе» (кроме заблокированных)

---

## В работе → На проверке

Проверяет: **исполнитель** перед передачей на проверку.

- [ ] Артефакт существует физически (файл сохранён в `output/`, ссылка рабочая)
- [ ] Все пункты чек-листа отмечены `- [x]` (ни один не пропущен)
- [ ] Написан итоговый комментарий к задаче:
  - что сделано (1–3 предложения)
  - на что обратить внимание при проверке
  - что осталось на усмотрение Жени
- [ ] Отклонения от исходного задания зафиксированы явно («изменил X потому что Y»)
- [ ] Результат содержит обязательные данные (если применимо):
  - [ ] Бюджет в ₽ (конкретная сумма, не «примерно»)
  - [ ] Даты в формате дд.мм.гггг
  - [ ] KPI — измеримый результат
  - [ ] Ответственный за следующий шаг
- [ ] Тон текста соответствует стандарту: нет запрещённых слов, нет канцелярита
- [ ] AI-арт/AI-изображения не предложены (абсолютный запрет)

---

## На проверке → Готово

Проверяет: **Женя Аржевский** или **Critic-агент**.

- [ ] Проверяющий явно написал «OK», «утверждаю», «принято» или эквивалент
- [ ] Все требования из исходной задачи выполнены
- [ ] Если задача возвращалась в «Не принято» — все замечания того цикла устранены
- [ ] Финальный файл находится в правильной директории:
  - Тактика: `output/tactic/`
  - Посты: `output/drafts/`
  - Исследования: `output/research/`
  - Отчёты: `output/reports/`
- [ ] Google Sheets обновлён (если задача связана с контентом или тендером)
- [ ] Результат не требует дополнительных действий для использования

**Важно:** «Почти принято» = «Не принято». Переход в «Готово» — только явное подтверждение.

---

## На проверке → Не принято

Проверяет: **Женя Аржевский** или **Critic-агент**.

- [ ] Написан конкретный комментарий: что именно не так (не «переделать», а «переделать X потому что Y»)
- [ ] Указан чёткий критерий для повторного принятия: «исправить X → снова на проверку»
- [ ] Если замечаний несколько — пронумерованы, чтобы исполнитель мог отмечать по одному
- [ ] Исполнитель подтвердил что понял комментарий (ответил или переместил в «В работе»)
- [ ] Дедлайн на доработку указан (если срочно)

---

## Не принято → В работе (возврат)

Проверяет: **исполнитель** перед повторным взятием.

- [ ] Все пункты из комментария «Не принято» прочитаны и поняты
- [ ] Чек-лист доработки составлен (по пунктам замечания)
- [ ] Если замечание касается фундаментального недопонимания задачи — уточнить у Жени перед стартом

---

## Дополнительно: критерии качества контента (для content-ops и writer)

Применять при переходе «В работе → На проверке» для любых текстов:

- [ ] Нет запрещённых слов: `уникальный`, `качественный`, `позволяет`, `является`, `данный`, `предоставляет`, `обеспечивает`, `в рамках`, `на сегодняшний день`, `в целях`, `осуществляется`
- [ ] Предложения ≤ 25 слов (длинные — признак AI-текста)
- [ ] Есть конкретные цифры (не «много», а «38 групп»)
- [ ] Тон: как другу за пивом, не официальный
- [ ] Нет упоминания AI-инструментов, нет AI-арта
- [ ] Master / Олег Грановский — только уважительно
